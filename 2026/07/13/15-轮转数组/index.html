轮转数组

题目描述

核心思路

让我们从最直觉的方法开始，一步步推到最优解。

第一反应：暴力模拟

如果没有任何技巧，我们第一反应会怎么做？

“轮转 k 次嘛，那就一次一次来——每次把最后一个元素保存下来，然后把前面的所有元素往后挪一个位置，最后把保存的元素放到最前面。”

听起来很直接，对吧？来算一下代价：每次轮转需要遍历整个数组（O(n)），一共要轮转 k 次，所以整体是 O(n × k)。如果 n 和 k 都很大，比如 n = 10^5、k = 10^5，那就要执行 10^10 次操作——在现代计算机上也是不可接受的。

有没有办法避免这种重复劳动？

第一步优化：用额外空间换时间

“如果允许我们用额外的数组，是不是可以一步到位？”

不妨想一下：每个元素最终会去到哪里？如果数组长度是 n，向右轮转 k 位，那么原来在位置 i 的元素应该去位置 (i + k) % n。这个映射关系很简单，我们只需要创建一个新数组，遍历一遍原数组，把每个元素放到它该去的位置就行了。

这样时间复杂度降到了 O(n)，但空间复杂度是 O(n)——我们多占用了和原数组一样大的空间。

题目要求我们能否做到 原地 修改？也就是空间复杂度 O(1)。这需要我们换个思路。

再思考：能否原地完成？

“不用额外数组，我们能直接交换元素的位置吗？”

一个自然的想法是：我们从位置 0 开始，把 nums[0] 放到它该去的位置 (0 + k) % n，然后原来那个位置的元素被挤出来了，我们拿着它继续去它该去的位置……这样会形成一个环。走完一个环之后，再从下一个位置开始。

这个过程叫 环状替换（cyclic replacement）。遍历所有元素，每个元素恰好被移动一次，时间 O(n)，空间 O(1)。

不过这个思路有一个细节需要处理：什么时候一个环结束？什么时候所有环都走完了？我们用一个计数器来跟踪已移动的元素个数，当移动了 n 个元素时就完成了。具体实现中，我们会多次启动替换——如果起点恰好在一个已经走过的环上，就跳过。

最优解：反转的魔法

“有没有更优雅的方式？既不需要计数，也不需要处理环的边界？”

有的。让我们观察一下轮转前后的变化规律。

仔细看：轮转后的数组可以看作 最后 k 个元素被放到了前面，前 n−k 个元素被移到了后面，而且两部分的内部相对顺序不变。

复杂度分析表

版本 时间复杂度 空间复杂度 说明
暴力模拟 O(n × k) O(1) 每次轮转遍历整个数组，轮转 k 次

三种 O(n) 解法中，三次反转法 因为代码最简洁、思路最直观，通常是面试中的首选。

延伸思考

这道题让我们学到了什么？

反转操作是一个强大的工具。有时候，看似复杂的轮转问题，可以通过”整体反转 + 局部反转”来优雅解决。背后的直觉是：轮转本质上是在一个环形结构上做平移，而反转是一种可以”重置部分顺序”的操作。当我们把两种操作组合起来时，就能达到平移的效果。

另外，取模简化（k %= n）也是一个常用技巧——处理循环移位问题时，先通过取模把大步长缩小到数组范围以内，可以避免很多边界处理的麻烦。

下次遇到类似问题优先观察什么？

  • 是不是环状结构上的平移？ 如果是数组轮转、字符串循环移位这类问题，优先想想反转法。
  • 两段内部有序、整体需要平移的结构，往往是反转法的适用场景。
  • 能不能原地做？ 如果要求 O(1) 额外空间，先排查物理结构上是否有对称性可以利用。

记住，算法题的最优解往往不是凭空想出来的，而是从暴力解出发，一步步问自己”哪里浪费了？能不能合并？有没有对称性？”，最终自然推导出来的。
