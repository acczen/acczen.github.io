LeetCode 189. 轮转数组（rotate-array）

题目

LeetCode 189. rotate-array

给定一个整数数组 nums，将数组中的元素向右轮转 k 个位置，其中 k 是非负数。

示例 1：

示例 2：

提示：

  • $1 \le nums.length \le 10^5$
  • $-2^{31} \le nums[i] \le 2^{31} - 1$
  • $0 \le k \le 10^5$

进阶：

  • 尽可能想出三种及以上的解决方案。
  • 你可以使用空间复杂度为 $O(1)$ 的原地算法解决这个问题吗？

核心思路

拿到这道题，我们不妨先问自己一个问题：“右移 k 步”到底意味着什么？能把这个问题换个角度看吗？

我们先从最直观的做法开始。

暴力模拟

第一反应大概是：模拟题目的描述，每次把整个数组右移 1 位，共做 k 次。每次”右移 1 位”就是保存最后一个元素，然后把前面所有元素依次后移一格，最后把保存的元素放到开头。

这个做法的时间是 $O(n \times k)$，在 $n$ 和 $k$ 都达到 $10^5$ 级别时会超时。瓶颈在哪？

瓶颈在于：我们一次只移动一位，做了太多冗余的搬运工作。有没有办法让每个元素一次性到达它该去的位置？

借助额外空间

如果我们不在乎空间，思路就非常简单：创建一个新数组，把 nums[i] 直接放到新数组的 (i + k) % n 位置上，最后拷贝回来。时间 $O(n)$，空间 $O(n)$。

但进阶要求我们尝试 $O(1)$ 空间的原地算法。那问题来了——

如果不让你借”外援”（额外数组），你能在数组内部直接完成这个”乾坤大挪移”吗？

一个关键的观察

让我们盯着示例的输入输出多看几眼：

你能发现什么规律？试试把数组分成两部分看：[1,2,3,4] 和 [5,6,7]。输出变成了 [5,6,7] + [1,2,3,4]——后 $k$ 个元素被提到了前面，前 $n-k$ 个元素被推到了后面。

这就引出了一个非常优雅的思路：反转。

反转一个数组，就是把顺序完全颠倒。那反转恰好能把”从哪切开”这件事办了吗？我们来看看：

  • 整体反转：[7, 6, 5, 4, 3, 2, 1]
  • 此时前 $k$ 个是 [7, 6, 5]，后 $n-k$ 个是 [4, 3, 2, 1]
  • 我们想要前 $k$ 个变成 [5, 6, 7]（正序），后 $n-k$ 个变成 [1, 2, 3, 4]（正序）
  • 分别反转前 $k$ 个和后 $n-k$ 个就行了！

用一个直观的图示：

这就是著名的三次反转法。它的美在于：不需要任何额外空间，每个元素恰好被访问常数次，总时间 $O(n)$。

细节补充：k 可能大于 n

还有一个小细节容易被忽略：如果 $k$ 大于数组长度 $n$ 怎么办？

想一想，一个长度为 $n$ 的数组轮转 $n$ 步后等于没动。所以实际上我们只需要轮转 $k \bmod n$ 步。预处理 $k = k\ %\ n$，如果 $k = 0$ 直接返回即可。

解法：三次反转

为什么选择这个解法？

方案 时间 空间 原地？
暴力模拟 $O(n \times k)$ $O(1)$ 是
额外数组 $O(n)$ $O(n)$ 否
环形替换 $O(n)$ $O(1)$ 是
三次反转 $O(n)$ $O(1)$ 是

环形替换（Cyclic Replacement）也是 $O(n)$ / $O(1)$ 的正解，但理解和实现都更复杂，需要处理”循环回到起点”的情况。三次反转法代码短、思路直观、不易出错，是面试中推荐的首选解法。

算法步骤

  1. 预处理 $k = k \ % \ n$，若 $k = 0$ 直接返回。
  2. 反转整个数组：[0, n-1]
  3. 反转前 $k$ 个元素：[0, k-1]
  4. 反转后 $n-k$ 个元素：[k, n-1]

反转函数用经典的双指针即可：一头一尾交换，向中间靠拢。

复杂度分析

维度 分析
时间复杂度 $O(n)$ — 每次反转恰好遍历区间内所有元素一次，三次反转总共访问约 $n + k + (n-k) = 2n$ 次元素，常数级别忽略，整体线性。
空间复杂度 $O(1)$ — 反转函数只用了几个临时变量来做交换，没有任何与 $n$ 相关的额外空间分配。

延伸思考

学到了什么？

轮转问题本质上是在做”分段重组”，而反转操作恰好能在 $O(1)$ 空间内完成任意位置的分段重排——“整体翻过来，再分别翻回去”这个技巧非常通用，比如反转字符串中的单词、轮转链表等场景都有类似的思想。

下次遇到类似问题，你会优先观察什么？

遇到需要把数组”切开再拼接”的问题时，优先考虑：反转操作能不能作为”不借助额外空间重组数组”的原子操作？ 具体来说，画一下”整体反转 + 分段反转”对数组结构的影响，看看能否把你的目标排列拆成这三步。
