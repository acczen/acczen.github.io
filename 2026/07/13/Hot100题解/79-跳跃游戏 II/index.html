跳跃游戏 II

跳跃游戏 II

核心思路

第一层：我们拿到这道题，先想什么？

题目给了我们一个非负整数数组 nums，每一个元素代表从该位置最多能向前跳几步。我们从下标 0 出发，问最少跳几次能到达最后一个位置。题目保证一定能跳到终点。

最原始的想法：枚举所有可能的跳法，取步数最少的那个。

暴力回溯 —— 尝试所有路径

对于当前位置 i，我们可以跳 1 到 nums[i] 步中的任意一种。每跳一次就递归到新位置，到达终点时记录步数，最终取最小值。

这个算法的时间复杂度是指数级的 $O(2^n)$，因为每个分支都在不断分叉。当数组长度达到 $10^4$ 时，这显然不可行。

我们不禁要问：问题出在哪？

大量的子问题被重复计算了。比如从位置 2 跳到位置 5，和从位置 3 跳到位置 5，后续的走法是完全一样的，但我们却分别递归去算。

第二层：加个备忘录，能不能行？

既然有重叠子问题，自然想到动态规划。

定义 dp[i]：从位置 i 跳到终点所需的最少步数。

那么我们要求的就是 dp[0]，递推关系也很直接：

$$dp[i] = \min_{1 \le step \le nums[i],; i+step < n} \big(1 + dp[i+step]\big)$$

边界：dp[n-1] = 0。

写出来长这样（自底向上）：

两层循环，外层 $O(n)$，内层最坏跳 $O(n)$ 步，总体 $O(n^2)$。空间 $O(n)$。

还能更快吗？

当 $n=10^5$ 时，$n^2$ 是 $10^{10}$ 次操作，仍然不可接受。我们需要想办法去掉内层的那层循环。

直觉告诉我们：我们其实不需要精确知道每个位置的最小步数，我们只需要在”不得不跳”的时候才跳，而且每次都要跳得最远。

这听起来像是贪心。

第三层：贪心 —— 像 BFS 一样”层”进

让我们换一种视角。把问题看作一个在数组上的 BFS：

  • 第 0 步，我们只能站在下标 0 这个位置，能到达的范围是 [0, nums[0]]。
  • 第 1 步，站在第 0 步能到达的任意一个位置，我们能到达的最远距离是多少？
  • 第 2 步，站在第 1 步能到达的任意一个位置，我们能到达的最远距离又是多少？
  • ……

每当我们把当前”层”能到达的所有位置走完，就必须迈出下一步（跳数加 1），并把下一步的边界更新为当前能到达的最远位置。

这完全等价于 BFS 的层序遍历。但因为我们不需要记录路径，只需要计数，所以连队列都不用，只用几个变量就够了。

具体做法：

  1. 维护三个变量：

    • currentEnd：当前这一步能到达的最远下标（当前层的边界）
    • farthest：遍历过程中，从已访问位置能到达的最远下标（下一层的边界）
    • jumps：已经跳了多少步

  2. 遍历数组 i 从 0 到 n-2（最后一个位置不用遍历，因为到了就结束了）：

    • 更新 farthest = max(farthest, i + nums[i])
    • 如果 i == currentEnd，说明当前层走完了，必须跳一步：
      • jumps++
      • currentEnd = farthest（把边界扩展到下一层能到达的最远位置）

  3. 返回 jumps。

为什么遍历到 n-2 就够了？

因为当 i 到达 n-1 时，我们已经站在终点了，不需要再跳。而题目保证能到达终点，所以遍历过程中一定会在某个时刻让 currentEnd >= n-1，此时循环可能提前结束，或者即使继续也不会影响结果。

这个算法的正确性怎么保证？

我们需要证明：每次在”当前层边界”处选择跳一步，并且把下一层边界设为当前能到达的最远位置，得到的步数一定是最少的。

可以用反证法或归纳法来理解：假设存在一个更优解，它在某一步跳到了一个比 farthest 更近的位置，那么这一步之后它能到达的范围一定不会比用 farthest 更远。因此，为了用最少的步数到达终点，每一步都应该尽可能扩展最远距离。这就是贪心选择性质。

举个例子走一遍

nums = [2, 3, 1, 1, 4]

i nums[i] farthest currentEnd jumps 说明
0 2 max(0, 0+2)=2 0 0 初始化
0 - - - - i==currentEnd，跳一步：jumps=1, currentEnd=2
1 3 max(2, 1+3)=4 2 1 更新 farthest
2 1 max(4, 2+1)=4 2 1 i==currentEnd，跳一步：jumps=2, currentEnd=4
3 1 max(4, 3+1)=4 4 2 更新 farthest，但 i < currentEnd，不跳

返回 2。实际路径：0 → 1 → 4，或者 0 → 1 → 3 → 4（但最少就是 2 步）。

延伸思考

这道题让我们学到了什么？

  1. 不要迷信 DP。 虽然 DP 是处理”最少步数”类问题的常用武器，但当状态转移需要遍历所有可能的步长时，$O(n^2)$ 的代价可能不是最优的。有时题目有更深的贪心结构。

  2. BFS 的另一种表现形式。 当我们需要”最少步数/层数”但不需要具体路径时，可以用几个变量模拟 BFS 的层次扩展，省掉队列的开销。这种技巧在很多”跳跃”类问题中非常通用。

  3. 贪心的关键在于”不得不”。 这里的贪心并不是每一步都选最远位置跳，而是在当前能触及的范围内先探索，等到不得不迈出下一步时，才选择能跳最远的那个方向。这种”延迟决策”的贪心比”立即决策”的贪心更稳健。

下次遇到类似问题，优先观察什么？

  • 是否能转化为 BFS 的层次遍历？ 只要问”最少步数/最短路径”，就想想 BFS。如果图的节点是线性的、边是动态计算的，很可能可以用类似的双指针贪心。
  • 题目是否保证有解？ 跳跃游戏 I 只问能否到达，跳跃游戏 II 问最少步数。保证有解时，贪心的边界处理会简单很多。
  • 状态转移是否连续？ 如果 dp[i] 需要从一段连续区间 [i+1, i+nums[i]] 中取最小值，可以考虑用线段树、单调队列等结构优化到 $O(n \log n)$ 或 $O(n)$。但在本题中，贪心的 $O(n)$ 是更简洁的答案。

举一反三

  • 跳跃游戏 I（LeetCode 55）：只判断能否到达终点，同样可以用贪心维护最远可达位置，如果遍历过程中 farthest < i 就返回 false。
  • 视频拼接（LeetCode 1024）：本质上也是”最少区间覆盖”问题，和本题的贪心 BFS 思路异曲同工。
  • 最少箭头引爆气球（LeetCode 452）、无重叠区间（LeetCode 435）：这些区间调度问题同样利用了贪心的排序 + 边界扩展思想。

跳跃游戏 II 是一个典型的”看上去像 DP，但贪心更美”的题目。从暴力到 DP 再到贪心 BFS，我们一步步拆掉了不必要的计算，最终得到了 $O(n)$ 时间、$O(1)$ 空间的简洁解法。希望下次你在面对”最少步数”问题时，能多一层”能不能 BFS 加贪心”的直觉。
