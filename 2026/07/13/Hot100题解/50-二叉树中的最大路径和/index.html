二叉树中的最大路径和

二叉树中的最大路径和

核心思路

从暴力解法开始思考

题目要求我们找出一条路径，使得路径上所有节点的值之和最大。路径可以从任意节点出发，到任意节点结束，只要路径上的节点是连通的，且每个节点最多出现一次。

我们先想最直接的做法：枚举所有可能的路径。

对于一个二叉树来说，有多少条路径？粗略一想，任意两个节点之间有一条唯一路径（通过最近公共祖先），所以路径的数量是 $O(n^2)$ 级别的。每一条路径我们还要计算和，那暴力就是 $O(n^3)$ 甚至更高——这显然不可接受。

但我们不妨从另一个角度思考：如果我们固定一个节点作为路径的”最高点”（即路径上深度最小的节点），会发生什么？

任意一条路径，在二叉树中都有一个唯一的”最高点”——也就是路径上离根最近的那个节点（LCA）。如果我们枚举每个节点作为最高点，然后计算”从该节点出发，往左子树能延伸的最大和”与”往右子树能延伸的最大和”，那么经过这个节点的最大路径和就是：

为什么是”最大贡献”而不是”最大路径和”？

好问题。这里的关键区别在于：当一条路径往上走去连接父节点时，它只能选择左子树或右子树中的一侧，不能两边都走——否则路径会分叉，就不连通了。

所以我们定义两个不同的概念：

  1. 贡献值（gain）：从当前节点出发，沿着一条不分支的路径往下走，能获得的最大和。它用于向上传递给父节点。
  2. 经过当前节点的最大路径和：左右子树贡献都取，形成一个”倒 V”形路径。它用于更新全局答案。

让我们把刚才的思路用递归语言精确描述。

问： 这个”路径和”和”贡献值”的区别能再说一下吗？

答： 用一个简单的例子：

在节点 20：

  • 左贡献 = 15，右贡献 = 7
  • 经过 20 的路径和 = 20 + 15 + 7 = 42（可以同时走两边，形成完整路径）
  • 贡献值 = 20 + max(15, 7) = 35（只能选一边，传给父节点 -10）

父节点 -10 拿着子节点的贡献值 35（右）和 9（左），自己算路径和 = -10 + 35 + 9 = 34，贡献值 = -10 + 35 = 25。

全局最大值是 42。

问： 路径可以不经过任何分支节点吗？比如只有一个节点？

答： 可以。如果二叉树只有一个节点 3，那么左贡献 = 右贡献 = 0（空节点贡献为 0），路径和 = 3 + 0 + 0 = 3，答案就是 3。我们的公式自然覆盖了这个情况。

问： 如果节点值是负数怎么办？比如整棵树都是负数？

答： 这也是为什么要和 0 取 max 的原因。子节点贡献为负，我们就不选它。但注意：节点本身可能是负数，但我们可以选择”只取这一个节点”——路径和公式中 node.val 是必加的，所以全局最大值至少会是整棵树中最大的那个负数。

举个例子：

在根节点 -2：左贡献 = max(-1, 0) = 0，右贡献 = max(-3, 0) = 0，路径和 = -2 + 0 + 0 = -2。但左子节点 -1 自己作为最高点时，路径和 = -1，更大。所以答案是 -1。我们的递归会遍历每个节点，所以不会漏掉。

算法的完整流程

  1. 初始化全局变量 maxSum = Integer.MIN_VALUE（因为节点值可能全为负）。
  2. 从根节点开始后序遍历（DFS）：
    • 递归计算左子树的贡献值 leftGain，如果为负则取 0
    • 递归计算右子树的贡献值 rightGain，如果为负则取 0
    • 经过当前节点的路径和 = node.val + leftGain + rightGain，用它更新 maxSum
    • 返回当前节点的贡献值 = node.val + max(leftGain, rightGain)
  3. 最后返回 maxSum。

后序遍历保证：在处理一个节点时，它的左右子树已经被处理完毕，子树的贡献值已经知道。这正是我们需要的自底向上的计算顺序。

这个算法的本质是在树上的动态规划——每个节点利用子问题的解（子树的贡献值）来构造当前问题的解。

复杂度分析

版本 时间复杂度 空间复杂度 说明
最优解（后序遍历） $O(n)$ $O(h)$ $n$ 为节点数，$h$ 为树高。每个节点恰好访问一次。空间来自递归调用栈，最坏 $O(n)$（链状树），平均 $O(\log n)$（平衡树）

暴力枚举的时间复杂度是 $O(n^2)$ 条路径 × $O(h)$ 计算和 = $O(n^3)$ 级别。我们的后序遍历 $O(n)$ 一次搞定，是理论最优解。

空间上，我们没有使用额外的数据结构，只用了递归栈。如果树退化成链表，递归深度会达到 $n$，需要注意栈溢出风险（不过这是所有递归算法的通病，可以改成迭代来规避）。

延伸思考

这道题让我们学到了什么？

第一，将”路径”的视角从”两端之间”切换到”最高点”。 这是解本题最关键的一步。在树上，任意路径都有一个唯一的最高点（LCA），这让我们可以把问题拆解成”在每个节点处汇聚左右子树的贡献”。

第二，区分”路径和”与”贡献值”。 前者用于更新全局答案，可以左右兼收；后者用于向上传递，只能取其一。这种”既要又要”的区分在树上问题中非常常见，比如求树的直径也是类似的思路。

第三，”负数即丢弃”的策略。 用 max(gain, 0) 来忽略负贡献，本质上是一种贪心——如果走入负收益的分支只会拖累结果，那不如不走。这个技巧在路径类问题中反复出现。

下次遇到类似问题优先观察什么？

  • 路径是否可以有分支？ 如果可以自由选择分支形态（如本题的”倒 V”形），考虑枚举”最高点”。
  • 路径方向是否受限？ 如果路径必须是自顶向下的（只能从父到子），问题会更简单，直接前缀和 + 哈希表就能解决。
  • 节点值是否可能为负？ 如果全正，路径越长越好；如果可能有负，就需要考虑”剪掉”负贡献。
  • 二叉树还是多叉树？ 本题解法可以自然推广到 N 叉树：从左右两个方向扩展为多个方向，取最大的两个贡献值相加即可。

换个角度：树的直径

如果你觉得本题有意思，不妨去看看 LeetCode 543. 二叉树的直径。那道题和本题是”同父异母”的兄弟——直径求的是路径上节点个数的最大值（或边的条数），而本题求的是节点值的和的最大值。思路完全一致：后序遍历，在每个节点处用 leftDepth + rightDepth 更新全局答案，返回 max(left, right) + 1。

一个关注”长度”，一个关注”和”，但解法骨架一模一样。理解了其中一个，另一个自然就会了。
