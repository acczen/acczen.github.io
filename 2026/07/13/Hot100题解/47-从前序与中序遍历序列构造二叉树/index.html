从前序与中序遍历序列构造二叉树

从前序与中序遍历序列构造二叉树

核心思路

题目：给定一棵二叉树的前序遍历和中序遍历结果，重建这棵二叉树。

前序遍历：[ 根, 左子树, 右子树 ]
中序遍历：[ 左子树, 根, 右子树 ]

这两句话，其实已经揭示了全部的答案。

第一步：最直觉的想法

已知一棵二叉树的前序遍历是 [3, 9, 20, 15, 7]，中序遍历是 [9, 3, 15, 20, 7]。我们要还原它。

不妨问自己：前序遍历的第一个元素是什么？

是根节点。因为前序遍历总是先访问根节点，再遍历左子树，最后右子树。所以前序序列的第一个元素 3 就是整棵树的根。

紧接着第二个问题来了：知道了根，左右子树怎么分？

这就轮到中序遍历登场了。在中序遍历中，根节点 3 的左边 [9] 是左子树的所有节点，右边 [15, 20, 7] 是右子树的所有节点。

那我们就知道：

  • 左子树的前序遍历是 [9]，中序遍历是 [9]
  • 右子树的前序遍历是 [20, 15, 7]，中序遍历是 [15, 20, 7]

问题规模变小了，但形式完全一样。 于是我们自然想到——递归。

第二步：暴力递归（朴素实现）

最直接的做法：每次从当前前序序列中取出第一个元素作为根，然后去中序序列中找到这个元素的位置，把两个序列分别切成左右两半，递归构建。

这个过程像不像切蛋糕？每次切一刀，分出左右两块，再对每块重复同样的操作。

这个思路完全正确，但每次递归都创建新的子数组，时间复杂度是 O(n^2)——每次找根的位置需要遍历中序，切割数组也有额外开销。

那么我们自然会想：能不能不创建新数组，用指针在原数组上操作？

第四步：哈希表加速中序查找

我们发现，每次找根节点在中序遍历中的位置是一个频繁操作。实际上我们可以提前把中序遍历中每个值对应的下标存到一个哈希表里，这样每次查找就是 O(1)。

valToIndex = { 9:0, 3:1, 15:2, 20:3, 7:4 }

这样一来，给定根节点值，瞬间就知道它在 inorder 中的位置，然后左子树的节点个数就是 index - inStart。

让我们把完整的逻辑串起来：

  1. 前序第一个元素是根
  2. 用哈希表查出根在中序的下标 index
  3. 左子树节点数 leftSize = index - inStart
  4. 左子树在前序中的范围：[preStart+1, preStart+1+leftSize)
  5. 左子树在中序中的范围：[inStart, index)
  6. 右子树在前序中的范围：[preStart+1+leftSize, preEnd)
  7. 右子树在中序中的范围：[index+1, inEnd)

递归下去，每一个节点恰好处理一次，整体 O(n)。

这其实就是分治思想的一次漂亮实践：找到当前问题的”分界点”，拆成两个规模更小的子问题，递归解决。

第五步：扩展——为什么前序 + 中序能唯一确定一棵树？

不妨追问一下：为什么非要前序 + 中序？只给前序不行吗？

前序告诉你根是谁，但不告诉你左右子树的分界线在哪。中序恰好提供了这个分界线。两者互补，缺一不可。

版本 时间复杂度 空间复杂度 说明
暴力递归（切数组） $$O(n^2)$$ $$O(n^2)$$ 每次递归都创建新数组，找根的位置需要遍历中序
指针 + 遍历查找 $$O(n^2)$$ $$O(n)$$ 不再创建新数组，但每次递归都要在中序中线性查找根

其中 n 是二叉树的节点数。

  • 时间复杂度 $$O(n)$$：每个节点恰好被创建一次，哈希表查询为 $$O(1)$$，整体线性。
  • 空间复杂度 $$O(n)$$：哈希表存储 n 个映射；递归栈深度在最坏情况下（单链树）为 $$O(n)$$。

延伸思考

这道题让我们学到了什么？

分治思想的典型应用。 当我们遇到”大问题和小问题结构相同”的场景时，第一反应就应该是递归 + 分治。这道题的核心就是把构建整棵树的问题，拆解成构建左子树和右子树两个子问题。

两种信息互补的价值。 单看前序或单看中序都无法确定树的结构，但两者结合就唯一确定了。这提醒我们：当一个问题缺少约束条件时，寻找第二个维度的信息来消除歧义，往往就是突破口。

用哈希表优化查找是经典套路。 但凡在遍历中需要频繁查询”某个值的位置”，都可以考虑提前建立映射。这是”空间换时间”最常见的应用场景。

下次遇到类似问题优先观察什么？

  • 如果题目给了两种遍历序列，先问自己：每种遍历能提供什么信息？
    • 前序/后序 → 根的位置
    • 中序 → 左右子树的划分
  • 能不能用指针代替数组拷贝？在递归中用下标范围来界定当前子问题，既干净又高效。
  • 有没有频繁的查找操作？建一个哈希表一劳永逸。

扩展思考

如果我们给的是 后序 + 中序，还能重建吗？可以的，思路完全对称——后序的最后一个元素是根，在中序中找到它，划分左右子树，递归即可。

如果给的是 前序 + 后序 呢？这就不能唯一确定一棵二叉树了——前序和后序都只告诉我们根的信息，缺少左右子树的划分依据。只有当树是”真二叉树”（每个节点要么有两个孩子，要么是叶子）时，前序+后序才能唯一确定。

递归的分治思想，配上哈希表的空间换时间，两道经典的遍历序列相互约束——这道题虽小，但蕴含的思想却值得反复品味。
