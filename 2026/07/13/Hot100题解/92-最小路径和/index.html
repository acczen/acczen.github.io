最小路径和

最小路径和

核心思路

第 1 步：暴力解法——所有路径走一遍

拿到这道题，我们先从最直观的想法开始。

问题：从左上角走到右下角，每次只能向右或向下，求路径和的最小值。

让我们想想暴力怎么做。从起点出发，每一步都有两个选择——向右或向下。如果我们把所有可能的路径都走一遍，计算每条路径的和，再取最小值，是不是就能得到答案？

当然可以。但这就像一棵二叉树，每层分支数翻倍——对于一个 m×n 的网格，总路径数是 $C(m+n-2, m-1)$。当 m=100, n=100 时，这个数字大到计算机无法在合理时间内算完。

可是，这里面有没有重复计算的部分呢？

答案是肯定的。比如从 (0,0) 到 (1,1)，先右再下和先下再右，最终到达同一个格子。如果我们每次都重新从起点走到这个格子，显然浪费了之前的计算结果。

第 2 步：记忆化搜索——把算过的结果存起来

既然存在重复子问题，我们自然想到：把已经算过的结果记下来，下次直接取用。

定义 dfs(i, j) 表示从 (i, j) 走到右下角的最小路径和。那么：

边界条件：当 i = m-1 且 j = n-1 时，dfs(i, j) = grid[i][j]；当 i 或 j 越界时，返回无穷大。

这样，每当我们算出一个格子的结果，就把它存进 memo 数组。下次再遇到同一个格子，直接取 memo 值，无需重新递归。

这样做了之后，每个格子只会被计算一次，时间复杂度降到了 O(m×n)，空间复杂度也是 O(m×n)。

能不能连递归的开销也省掉？

好问题。既然我们已经想清楚了状态转移方程，不妨试试从底向上迭代计算。

第 3 步：动态规划——从底向上填表

我们翻转一下思路：不从起点往终点递归，而是从起点往终点迭代。

定义 dp[i][j] 表示从 (0,0) 走到 (i,j) 的最小路径和。

起点：dp[0][0] = grid[0][0]

第一行（只能从左边来）：dp[0][j] = dp[0][j-1] + grid[0][j]

第一列（只能从上边来）：dp[i][0] = dp[i-1][0] + grid[i][0]

其他位置：dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]

最终答案就是 dp[m-1][n-1]。

这样我们用一个二维 DP 表，从左上角开始一行一行地填，每个格子只依赖它左边和上边的格子，完全避免了递归调用。

时间复杂度 O(m×n)，空间复杂度 O(m×n)。

观察一下填表顺序，每一行只依赖上一行和当前行的左边——我们真的需要整个二维表吗？

第 4 步：空间优化——一维数组就够了

当我们填第 i 行时，dp[i][j] 只依赖于：

  • dp[i-1][j]——上一行的同一列（即旧值）
  • dp[i][j-1]——当前行的前一列（即新值）

所以，我们只需要一个长度为 n 的一维数组 dp，在遍历每一行时原地更新：

这里 dp[j] 更新前存的是上一行的值（即 dp[i-1][j]），dp[j-1] 已经在当前行被更新为 dp[i][j-1]。

空间复杂度从 O(m×n) 降到了 O(n)。

如果面试官说”你可以直接在原数组上修改”，空间还能省吗？

当然——连 O(n) 的额外空间都可以省掉，直接在 grid 上原地修改即可。不过需要和面试官确认是否可以修改输入数组。

延伸思考

这道题让我们学到了什么？

路径类 DP 的通用模式。

当我们遇到”从左上到右下，只能向右/向下”这样的约束时，问题的结构本身就决定了重叠子问题的存在——因为到达某个格子的路径是有限的，而后面的路径又依赖于前面的选择。

这种”一个方向推着另一个方向走”的依赖关系，天然适合动态规划。

更具体来说，我们学到了：

  1. 从暴力开始思考——不要一开始就想最优解，从能想到的最简单方法出发
  2. 识别重复子问题——判断问题的结构是否会产生重叠的计算
  3. 空间压缩的观察角度——填表时观察当前行依赖哪些行，能否滚动更新

下次遇到类似问题优先观察什么？

  • 移动方向：只能向右和向下 → 拓扑序天然存在，沿着行或列遍历即可
  • 依赖关系：dp[i][j] 依赖 dp[i-1][j] 和 dp[i][j-1] → 可以空间压缩
  • 是否可以原地修改：如果不禁止修改输入数组，可以省掉全部额外空间

类似的题目还有：

  • 62. 不同路径（计数版）
  • 63. 不同路径 II（带障碍）
  • 120. 三角形最小路径和

这三道题的思维链几乎一模一样，区别只在于转移方程中的操作：计数、条件判断、求和取 min。把最小路径和吃透，其他几道自然也会了。
