核心思路

题目：编写一个高效算法，在 m x n 矩阵中判断目标值是否存在。矩阵每行从左到右升序排列，且每行的第一个元素大于上一行的最后一个元素。

复杂度：O(mn)，没有利用任何排序信息。

这当然能通过小规模测试，但一旦矩阵变大，比如 1000 x 1000，最坏情况下要检查 100 万个格子。显然，我们需要利用”有序”这个条件。

追问：有序 + 查找，我们想到了什么？

看到”有序数组”和”查找”这两个关键词，我们脑海中会立刻响起一个声音——二分查找。

但问题是，这不是一维数组，是二维矩阵。怎么把二分法用到二维上来？

不妨想一下：如果我们能定位出目标值可能在哪一行，问题就退化成了”在一维有序数组中查找”。

让我们顺着这个思路往下走。

方案一：先定位行，再定位列

条件说：每行的第一个元素大于上一行的最后一个元素。

这意味着什么？我们看每行的首元素 matrix[i][0]，它们本身就构成一个递增序列！

所以我们可以：

  1. 对首元素列做二分查找，找到最后一个 <= target 的行
  2. 在该行内对整行做二分查找

这样就是 两次二分查找，复杂度 O(log m + log n)。

这个思路已经不错了，但还有更优雅的做法吗？

方案二：把矩阵”展平”

让我们重新审视题目的条件——每行有序，且行与行之间也是有序衔接的。实际上，这个矩阵本质上就是一个一维有序数组被”切”成了若干行。

那么，我们能不能直接对这个”虚拟的一维数组”做一次二分查找？

关键问题在于：给定一维下标 mid，如何映射回二维矩阵的 (row, col)？

不妨想一下：

  • row = mid / n（n 是列数）
  • col = mid % n

这样，整个矩阵就被我们”压平”了，而二分查找的对象是整个 m * n 个元素的有序序列。

这就是这道题最精妙的解法——一次二分，全量解决。

方案三：从右上角出发的”走迷宫”

还有一个有趣的视角。我们从矩阵的右上角出发：

  • 如果当前值 > target，说明目标在当前行左侧，列指针左移
  • 如果当前值 < target，说明目标在下一行，行指针下移
  • 如果相等，返回 true

这个方法的复杂度是 O(m + n)，不是最优，但思路非常直观，像在走一个迷宫——每一步都能排除一行或一列。

延伸思考

这道题让我们学到了什么？

降维思维。很多看似是二维的问题，只要数据满足某种有序性或连续性，就可以”压平”成一维来处理。一维能用的算法（排序、二分、双指针）比二维多得多。

举个例子，这道题的矩阵条件——每行有序，行间也有序——本质上就是”二维数组按行展开后是一个有序的一维数组”。识别出这个结构，解法就呼之欲出了。

下次遇到类似问题优先观察什么？

  1. 数据是否有序？ 有序是二分查找的入场券。
  2. 数据之间是否有单调性关联？ 比如这道题，行首元素本身构成递增序列。
  3. 能否降维？ 如果二维数据满足某种顺序，不妨问自己：把它拉直成一维，问题会不会变简单？

还有一个通用技巧：mid / colCount 和 mid % colCount 是处理二维矩阵时非常实用的映射关系，未来在类似题目（比如搜索二维矩阵 II、螺旋矩阵等）中也会反复用到。
