寻找重复数

寻找重复数

核心思路

场景还原

给定一个包含 n + 1 个整数的数组 nums，其中每个整数都在 [1, n] 范围内。数组中至少存在一个重复的整数，假设只有一个重复的整数，请找出这个重复的数。

限制：不能修改原数组，且只能使用 O(1) 的额外空间。

第一反应：暴力遍历

拿到这道题，最朴素的想法是什么？双层循环，外层固定一个数，内层扫描后面有没有相同的：

时间复杂度 O(n²)，空间 O(1)。但 n 可能到 10⁵ 甚至更大，O(n²) 显然撑不住。

能不能用哈希表？

暴力太慢，我们自然会想到用 HashSet 记录已经见过的数：

时间 O(n)，空间 O(n)。空间不符合”只能用 O(1) 额外空间”的限制。

那排序呢？排序后相邻元素相同就是重复数，但排序会修改原数组（题目禁止），而且即使允许，快速排序也需要 O(log n) 的递归栈空间。

看来需要一种既不依赖额外空间、又不修改数组的方法。

画风突变：把数组看作链表

这个限制让我们停下来重新审视题目条件——数组长度为 n+1，每个元素的值都在 [1, n] 之间。

每一个元素的值，恰好可以当作下标来使用。

让我们做一个小实验。把数组 [1, 3, 4, 2, 2] 画出来，以 nums[i] 作为从 i 出发指向的下一个位置：

发现了吗？这形成了一条链表：0 → 1 → 3 → 2 → 4 → 2 → 4 → 2 → …

因为存在重复数字 2，两个不同的位置（下标 3 和 4）都指向了同一个下标 2，于是链表里出现了环！

更一般地：

  • 数组长度 n+1，值域 [1, n]，根据鸽巢原理，至少有一个值出现两次。
  • 从下标 0 出发（0 不在值域 [1, n] 中，所以 0 不可能是任何节点的目标，0 是唯一的入度为 0 的点），一路沿着 nums[i] 跳转。
  • 由于存在重复值，必然有两个不同的下标指向同一个下标，从而形成环。
  • 重复的那个数，就是环的入口处下标的值。

问题转化为：在链表中找到环的入口。

Floyd 判圈算法

Floyd 判圈（龟兔赛跑）算法刚好能解决这个问题：

  1. 第一阶段：检测是否有环。慢指针每次走一步，快指针每次走两步。如果存在环，它们一定会在环内相遇。
  2. 第二阶段：找到环入口。让一个指针回到起点（0），另一个留在相遇点。两个指针每次都走一步，它们再次相遇的位置就是环的入口。

为什么第二阶段能定位到入口？让我们来推导。

设：

  • 从起点到环入口的距离为 a
  • 环入口到相遇点的距离为 b
  • 相遇点到环入口的剩余距离为 c（即环长 L = b + c）

慢指针走过的距离：a + b
快指针走过的距离：a + b + k·L（快指针可能绕了 k 圈）

因为快指针速度是慢指针的 2 倍：
2(a + b) = a + b + k·L
a + b = k·L
a = k·L - b = (k-1)·L + c

从起点走 a 步到达入口；从相遇点走 a 步也到达入口（因为相遇点走 c 步到入口，再走 (k-1) 圈回到入口）。所以让一个指针从起点出发，另一个从相遇点出发，每次一步，相遇处就是入口。

聊聊二分查找思路

在得到最优解之前，还有一种值得一提的解法——按值域二分。

假设重复数是 x，那么对于任意 mid：

  • 统计数组中 ≤ mid 的数的个数 cnt
  • 如果 cnt > mid，说明重复数在 [1, mid] 中
  • 否则在 [mid+1, n] 中

每次统计需要 O(n)，二分需要 O(log n)，总时间 O(n log n)，空间 O(1)。

虽然不如 Floyd 判圈的 O(n)，但思路不同、也很有启发——它用计数的方式利用了鸽巢原理。

延伸思考

这道题让我们学到了什么？

  1. 数组可以看作链表——当元素的值域与下标范围重合时，不妨把值当作”指针”。这是一个重要的建模技巧，很多”不可变数组找重复/缺失”的问题都可以向这个方向思考。

  2. Floyd 判圈算法不仅仅用于链表。它的本质是检测函数迭代序列中的循环，可以被应用到状态机、伪随机数生成器检测等领域。

  3. 在约束中寻找突破口。”不能修改数组 + O(1)空间”这个约束很强，但值域 [1, n] 和长度 n+1 这一对数字本身就是一个强烈的信号——鸽巢原理保证重复存在，也暗示了可以用下标做文章。

下次遇到类似问题优先观察什么？

  • 数据范围：长度和值域的关系——如果长度比值域大 1，大概率存在重复/缺失，可以思考下标映射。
  • 操作限制：不能修改数组 → 考虑值本身作为索引；O(1) 空间 → 排除哈希表。
  • 是否存在映射关系：值能否自然地映射回下标？如果能，链表 + 判圈这个工具箱就可以拿出来了。
  • 二进制视角：对于某些变种（如每个数出现奇数次/偶数次），异或运算也是 O(1) 空间的好选择。

Floyd 判圈解法是巧妙的，但更值得记住的是它背后的思维过程：当常规路径被堵死时，换一个视角重新描述问题——数组不是只能线性扫描，它也可以是一条隐形的链表。
