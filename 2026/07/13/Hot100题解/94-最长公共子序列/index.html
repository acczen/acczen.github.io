最长公共子序列

最长公共子序列

核心思路

从直觉出发

题目要求我们计算两个字符串的 最长公共子序列（LCS） 的长度。子序列不要求连续，只要求顺序一致。

不妨先想一个最简单的例子：text1 = "abc"，text2 = "ac"。肉眼观察，公共子序列可以是 "ac"，长度为 2。那如果字符串变长呢？我们如何让计算机来做这件事？

暴力搜索：尝试所有子序列

一种最直接的想法是：枚举 text1 的所有子序列，检查它是否也是 text2 的子序列，然后取最长的那个长度。

但一个长度为 m 的字符串有 (2^m) 个子序列。当 m = 1000 时，宇宙毁灭了都算不完。显然这个方向走不通。

自问：能否拆解成更小的问题？

让我们换个角度想：两个字符串的公共子序列长度，和它们各自的前缀子串的公共子序列长度，有没有关系？

假设我们用 dp[i][j] 表示 text1 的前 i 个字符（即 text1[0..i-1]）和 text2 的前 j 个字符（即 text2[0..j-1]）的最长公共子序列长度。

考虑 text1[i-1] 和 text2[j-1] 这两个字符：

  • 如果它们相等：那太好了！这个字符一定可以成为公共子序列的一部分。于是问题缩减为：text1 的前 i-1 个字符和 text2 的前 j-1 个字符的 LCS 长度，再加 1。

  • 如果它们不相等：那这两个字符不可能同时出现在公共子序列的末尾。此时，最长公共子序列要么来自 text1 的前 i-1 个字符和 text2 的前 j 个字符的 LCS，要么来自 text1 的前 i 个字符和 text2 的前 j-1 个字符的 LCS——取较大者。

看起来我们找到了递推关系！

状态转移方程

[
dp[i][j] = \begin{cases}
dp[i-1][j-1] + 1, & \text{if } text1[i-1] = text2[j-1] \
\max(dp[i-1][j], dp[i][j-1]), & \text{if } text1[i-1] \neq text2[j-1]
\end{cases}
]

边界条件也很自然：只要 i = 0 或 j = 0，其中一个字符串是空串，LCS 长度必为 0。

模拟算一下

以 text1 = "abcde"，text2 = "ace" 为例，我们填一下 dp 表：

dp “” a c e
“” 0 0 0 0
a 0 1 1 1
b 0 1 1 1
c 0 1 2 2
d 0 1 2 2
e 0 1 2 3
  • dp[1][1]：’a’ = ‘a’，所以 dp[1][1] = dp[0][0] + 1 = 1
  • dp[3][2]：’c’ = ‘c’，所以 dp[3][2] = dp[2][1] + 1 = 2
  • dp[5][3]：’e’ = ‘e’，所以 dp[5][3] = dp[4][2] + 1 = 3

最终答案是 3，即 "ace"。

能否优化空间？

观察 dp 表，dp[i][j] 只依赖三个位置：dp[i-1][j-1]（左上角）、dp[i-1][j]（正上方）、dp[i][j-1]（正左方）。

这意味着我们不需要存整个二维表。只需要两行（当前行和上一行）就够了，甚至可以用一个一维数组加一个额外变量来保存左上角的值。

延伸思考

这道题让我们学到了什么？

「两个序列的最值问题，优先想 DP」。当问题涉及两个序列的匹配、比较时，二维 DP 是最自然的思考框架。状态定义往往是 dp[i][j] 表示两个前缀子问题的结果，然后分末尾字符相等和不相等两种情况讨论。

下次遇到类似问题优先观察什么？

  • 子序列还是子数组？ 子序列不要求连续，子数组要求连续——后者往往可以用滑动窗口或一维 DP 解决。
  • 两个序列还是一个序列？ 两个序列的问题常用二维 DP（如编辑距离、两个字符串的删除操作）；一个序列的问题（如最长递增子序列）往往是一维 DP。
  • 可否空间优化？ 观察当前状态依赖哪些方向的状态，如果只依赖上、左、左上，就可以用滚动数组压到一维。

掌握了 LCS，后面遇到「编辑距离」「两个字符串的最小 ASCII 删除和」等题目时，你会发现它们其实都源自同一个思考模板——你已经有了解题直觉。
