最大子数组和

最大子数组和

题目：给你一个整数数组 nums，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

例如：nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]，输出 6，因为子数组 [4, -1, 2, 1] 的和最大。

核心思路

第一反应：暴力枚举所有子数组

如果没有任何技巧，我们第一反应会怎么做？

不妨想一下，”连续子数组”就是数组中某一段区间 [i, j]（0 ≤ i ≤ j < n）。要找到最大和，最直接的办法就是列举所有可能的 (i, j) 对，分别计算每一段的和，然后取最大值。

三层循环：i 枚举起点，j 枚举终点，k 累加区间。时间复杂度 O(n³)，空间复杂度 O(1)。

这个解法的问题在哪里？显然——大量的重复计算。每次计算 sum(i, j)，我们都把从 i 到 j 的数重新加了一遍，但 sum(i, j) 明明等于 sum(i, j-1) + nums[j]，我们却没有利用这个关系。

第一次优化：去掉最内层循环

两层循环，时间复杂度 O(n²)。好了一些，但 n 如果到 10⁵ 级别，O(n²) 仍然不可接受。

还能不能更快？—— 想想有没有更本质的观察

我们停下来想一想：上面的枚举策略本质上是在”猜”起点和终点。但一个最优解一定有一个起点和一个终点，有没有办法不以枚举起点的方式来思考？

不妨换一个视角：如果固定子数组的终点为位置 j，我们想要的是”以 j 结尾的最大子数组和”。如果把这个值记作 dp[j]，那么全局答案就是 max(dp[0], dp[1], ..., dp[n-1])。

但 dp[j] 和 dp[j-1] 之间有没有关系？我们来想：

  • 以 j 结尾的子数组，要么只包含 nums[j] 自己，要么是以 j-1 结尾的最大子数组再加上 nums[j]。
  • 没有其他可能，因为必须连续，且必须包含 j。

于是：
[
dp[j] = \max(nums[j], ; dp[j-1] + nums[j])
]

这就是动态规划的思路！dp[j] 只依赖 dp[j-1]，所以我们可以用一个变量滚动更新。

时间复杂度 O(n)，空间复杂度 O(1)。

这个算法有一个响亮的名字——Kadane 算法。

背后的直觉

为什么这个公式是对的？我们再从直觉上理解一下。

假设我们正从左到右扫描数组，手里拿着”当前子数组的和”。

  • 如果当前和已经是一个负数，那么下一个数字与其接在后面，不如自己另起炉灶——因为负数和只会拖累后面的数字。
  • 如果当前和是正数，那它对后面的数字是有”贡献”的，把它带上没有坏处。

这就是 dp = max(nums[j], dp + nums[j]) 这句代码在做的事情：在”接上前面的”和”重新开始”之间做选择。

另一种视角：分治法

如果面试官说”试试用分治做”，我们也可以从分治的角度理解这个问题。

把数组从中间分成两半，那么最大子数组和必然出现在三种情况之一：

  1. 完全在左半边
  2. 完全在右半边
  3. 跨越中点

前两种情况递归下去就行，第三种情况需要从中间向两边扩展，分别向左和向右找最大后缀和和最大前缀和，然后加在一起。

分治版的时间复杂度是 O(n log n)（主定理 T(n) = 2T(n/2) + O(n)），空间复杂度 O(log n) 来自递归栈。

不过分治并不是本题的最优解，但它是一个很好的思维拓展——很多区间类问题都可以用分治的思路来解。

延伸思考

这道题让我们学到了什么？

状态定义比转移方程更重要。 这道题最精彩的转化不是公式本身，而是把”全局最大子数组”拆解成了”以每个位置结尾的最大子数组”——通过固定终点，把一个看似需要枚举起点和终点的问题，简化成了一维的递推。这种”固定一个维度，考虑以某元素结尾/开头”的思路，在子数组、子序列问题中反复出现（比如最长递增子序列）。

另外，Kadane 算法的代码只有短短几行，但背后是动态规划思想的体现。有时最优解看起来很简单，但推导它的思考过程才是真正有价值的东西。

下次遇到类似问题优先观察什么？

  1. 问题是否涉及”连续子区间”或”子数组”？如果是，试试固定结尾的 DP 思路。
  2. 数据范围暗示了什么？ n ≤ 10⁴ 可能 O(n²) 还能接受，n ≥ 10⁵ 就必须 O(n) 或 O(n log n)。
  3. 有没有”接上一个 vs. 重新开始”的二选一决策？ 这往往是 Kadane 类问题的信号。
  4. 如果 DP 太简单，面试官追问其他方法，分治可作为思维深度的体现——虽然不是最优，但展示了多角度理解问题的能力。
