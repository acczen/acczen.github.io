打家劫舍

题目：你是一个专业小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金。唯一限制你的是，相邻的房屋装有连通的防盗系统，如果同一晚上有两间相邻的房屋被盗，系统会自动报警。

给定一个代表每间房屋存放金额的非负整数数组 nums ，请计算 不触动警报装置的情况下，一夜之内能够偷窃到的最高金额。

例如：nums = [1, 2, 3, 1]，输出 4（偷 1 号 + 3 号，金额 1 + 3 = 4）。

核心思路

从「暴力枚举」开始：每个房子，偷还是不偷？

我们先从一个朴素的想法开始。

对于每一间房子，我们只有两个选择：偷 或者 不偷。如果偷了当前这间，那么下一间就不能偷；如果不偷，下一间可以自由选择。

用自然语言描述决策过程：

从第 0 间房子开始，我们有两种分支：偷（跳到第 2 间）或不偷（走到第 1 间）。对每种分支继续做同样的选择，直到走完所有房子。最后取所有分支中累计金额最大的那个。

这其实就是一棵二叉决策树——每层对应一个房子，左分支代表偷，右分支代表不偷。树的高度是 n，叶子节点数是 2^n。

那么问题来了——这棵递归树里，有没有重复计算？

发现重复子问题：「后面的事情」是一样的

让我们画一个小规模的例子：nums = [1, 2, 3, 1]。

注意到 robFrom(3) 出现了多少次？——三次。robFrom(2) 出现了两次。robFrom(4) 反复出现。

这揭示了一个关键性质：最优子结构。robFrom(i) 的返回值只取决于 i，而不取决于我们是怎么到达 i 的。一旦计算过 robFrom(3)，下次再遇到就应该直接取结果，而不是重新展开整棵树。

这就是动态规划的核心思想——记住已经算过的子问题。

这是一个质的飞跃——从指数级降到了线性级。

改为递推：自底向上的 DP

既然子问题是从大 i 到小 i 依赖的（robFrom(i) 依赖 robFrom(i+1) 和 robFrom(i+2)），我们也可以反过来，从后往前填表：

定义 dp[i] 表示从第 i 间房子开始偷（即考虑 nums[i..n-1]），能获得的最大金额。

用表来描述就是这样的递推过程（从右向左）：

i（从右向左） nums[i] dp[i+2] dp[i+1] dp[i] = max(nums[i] + dp[i+2], dp[i+1])
3 1 0 0 max(1+0, 0) = 1
2 3 0 1 max(3+0, 1) = 3
1 2 1 3 max(2+1, 3) = 3
0 1 3 3 max(1+3, 3) = 4

答案 dp[0] = 4，与之前一致。

当然，我们也可以从前往后填。不妨换一个角度定义：

定义 dp[i] 表示 到第 i 间房子为止（考虑 nums[0..i]），能偷到的最大金额。

递推关系是什么呢？

  • 如果偷第 i 间，那么第 i-1 间不能偷，最大金额是 nums[i] + dp[i-2]
  • 如果不偷第 i 间，那么最大金额就是 dp[i-1]

所以：

初始条件：

  • dp[0] = nums[0]
  • dp[1] = max(nums[0], nums[1])

两种方向殊途同归，都是 O(n) 时间、O(n) 空间。

空间压缩：O(1) 的极致

仔细看递推式：dp[i] = max(nums[i] + dp[i-2], dp[i-1])。

延伸思考

这道题让我们学到了什么？

1. 动态规划就是「带缓存的暴力」

暴力枚举之所以慢，是因为它没有「记忆」——同一个子问题被反复求解。一旦我们把子问题的结果存下来，指数就变成了线性。这个转变非常优美，而且几乎不需要额外的数学知识，只需要我们能识别出重复的子问题。

2. 空间压缩是可选的锦上添花，不是必须

很多同学一上来就追求 O(1) 空间的写法，反而把自己绕晕了。不妨先写出最清晰的 DP 数组版本，把逻辑跑通，再看能否压缩。正确性优先，优化其次。

3. 状态定义决定了递推方向

「从第 i 间往后看」和「看到第 i 间为止」两种定义方式都是对的，但递推方向不同。理解状态定义比背公式重要得多——定义清楚了，递推关系自然就浮出水面。

下次遇到类似问题优先观察什么？

  • 能不能抽象成「每个元素选或不选，且相邻有约束」？ 这是打家劫舍类问题的标志。只要看到”不能相邻”、”不能连续”这类字眼，优先考虑 DP。
  • 依赖关系是什么？ 打家劫舍的依赖是一维的、线性的，所以只用两个变量就够了。如果依赖更复杂（比如二维网格、树形结构），可能需要更复杂的 DP 形式。
  • 状态可以压缩吗？ 先写出完整 DP，再观察递推公式中到底用到了前几项。只用到了前两项 → 两个变量；只用到了前一项 → 一个变量；用到了全部 → 老老实实开数组。
