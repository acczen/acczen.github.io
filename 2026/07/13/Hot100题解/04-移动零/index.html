移动零

移动零

核心思路

第一反应：暴力法 —— 用额外空间换时间

如果没有任何技巧，我们第一反应会怎么做？很简单：创建一个新数组，把非零元素按顺序拷贝过去，剩下的位置补零。

这个解法清晰直观，时间复杂度 O(n) 也很优秀。但问题在于它使用了 O(n) 的额外空间。

题目要求我们在原地修改数组，这意味着我们只能用常数级的额外空间。

于是我们不得不思考：能不能在不借助额外数组的情况下完成这个操作？

第一次优化：双指针 —— 先移动，再补零

不妨想一下：把零”移动”到末尾，本质上是把所有非零元素按顺序挤到前面。既然如此，我们能不能用一个指针来记录”下一个非零元素该放的位置”？

答案是肯定的。这就是双指针的雏形：

  • 用一个指针 pos 表示下一个非零元素应该存放的位置
  • 遍历数组，每当遇到一个非零元素，就把它放到 pos 位置，然后 pos++
  • 遍历结束后，从 pos 到数组末尾的所有位置全部赋值为 0

这个思路很自然——先处理非零元素，再统一补零。整个过程只需要一次遍历加一次填充，时间复杂度 O(n)，空间复杂度 O(1)。

不过这里面有一个小细节：如果数组开头就是非零元素，我们会把它赋值到自己的位置上（nums[0] = nums[0]），这是多余的赋值。但这不是大问题，不影响正确性。

再进一步：用交换代替赋值+补零

有经验的读者可能会注意到，上面的方法需要两次遍历（一次移动非零，一次填充零）。能不能一次遍历就搞定？

让我们换个角度思考。我们可以维护一个指针 j，它始终指向第一个可能为 0 的位置（初始时 j = 0）。然后遍历数组：

  • 当 nums[i] != 0 时，我们交换 nums[i] 和 nums[j]，然后 j++

为什么这样可以工作？因为 j 始终指向已处理区间中第一个非零元素应该去的位置，而 i 在 j 前面或与 j 相同。当 i 和 j 之间出现零时，j 就会停在零的位置上，等待下一个非零元素来交换。

画个图帮助理解：

通过交换，我们保证 j 左边的所有元素都是非零的，而 j 及其右边的元素都还没有被处理。这样一次遍历就完成了零的移动，而且不需要额外的补零操作。

这个解法还有一个隐藏的好处：如果数组里根本就没有零，那么 i 和 j 会同步前进，每个元素和自己交换（相当于没有操作），性能上也没有浪费。

最终核心思想

用双指针将数组逻辑分区：[0, j) 是已处理的非零区域，[j, i) 是已处理但被跳过的零，[i, n) 是未处理区域。每次遇到非零元素，就把它交换到 j 位置，扩大已处理的非零区域。

这是一种典型的”分区思想”——类似于快速排序中的 partition 过程。我们不是在”移动零”，而是在筛选非零元素到前面，零自然就被留在了后面。

复杂度分析

版本 时间复杂度 空间复杂度 说明
暴力法（额外数组） O(n) O(n) 遍历一次数组拷贝非零元素，额外数组存放结果
双指针（两次遍历） O(n) O(1) 第一遍移动非零到前面，第二遍将剩余位置补零
双指针（一次遍历 + 交换） O(n) O(1) 遍历一次，遇到非零就交换到前面，零自动归位

三种解法的时间复杂度都是 O(n)，但空间复杂度上双指针法明显更优。最优解法不仅满足原地修改的要求，还只用了一次遍历，代码也最为精简。

从实际运行角度看，虽然三种解法的时间复杂度相同，但最优解避免了第二次遍历的常数开销，在大数组上会有更好的实际表现。

延伸思考

这道题让我们学到了什么？

  1. 双指针的分区思想：用两个指针将数组划分为不同的逻辑区域，各自维护不同的语义。这种思想在数组问题中极其常见——从快速排序的 partition，到有序数组的去重，再到这道题的零元素移动，本质上都是同一个模式。

  2. 从”被动移动”到”主动筛选”：我们一开始想的是”把零往后移”，这是被动的视角。而最优解换了一个角度——“把非零往前筛”，这使问题变得简单。在算法中，转换视角往往比直接硬碰硬更有效。

  3. 交换代替赋值+补零：通过交换操作，一次遍历就同时完成了非零元素的前移和零元素的后置，减少了不必要的操作。

下次遇到类似问题优先观察什么？

  • 原地修改的限制：一旦题目要求原地操作（in-place），就要立刻考虑是否需要使用双指针
  • 保持相对顺序：当题目要求保持元素的相对顺序时，通常意味着我们不能简单地排序或重排，而是需要稳定的操作方式
  • 分区思想：如果问题涉及将数组中的元素按照某种条件分成两部分（比如大于/小于某个值、奇数和偶数、非零和零），可以想一下是否能用双指针扫描+交换的方式解决

这道题虽然简单，但它蕴含的双指针和分区思想是算法大厦的基石。把这道题吃透了，再遇到”奇偶分离””正负分离”等类似题目时，我们就能一眼看穿本质。
