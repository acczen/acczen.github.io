杨辉三角

杨辉三角

核心思路

题目：给定一个非负整数 numRows，生成「杨辉三角」的前 numRows 行。

示例：

第一步：最直观的想法是什么？

看到”杨辉三角”这个概念，我们不妨想一下它的数学定义。

杨辉三角（也称帕斯卡三角）的严格数学定义是：第 n 行第 k 个数字等于组合数 $C(n, k)$，即从 n 个不同元素中取 k 个的组合数：

$$C(n, k) = \frac{n!}{k!(n - k)!}$$

那么最直接的做法就是：遍历每一行、每一列，用组合数公式计算出每个位置的值。

这能解决问题吗？当然能。但每计算一个值都要算三个阶乘，代价太大了。我们停下来想一想：有没有更简单的性质可以利用？

第二步：杨辉三角最关键的性质是什么？

让我们观察一下杨辉三角的结构：

有没有发现一个规律？每个数字等于它上方两个数字之和。

用数学语言说就是：

$$dp[i][j] = dp[i-1][j-1] + dp[i-1][j]$$

其中 $dp[i][j]$ 表示第 i 行第 j 列的值（从 0 开始索引），而 $dp[i-1][j-1]$ 和 $dp[i-1][j]$ 就是它左上角和正上方的两个数。

我们还需要注意边界条件：每行的第一个和最后一个数字总是 1。

这个性质比组合数公式简洁太多了。这样一来，我们完全不需要计算阶乘，只需要从第一行开始，逐行推导即可。

第三步：如何实现”逐行构建”？

我们来模拟一下这个过程：

  • 第 0 行：[1] — 直接放进去。
  • 第 1 行：[1, 1] — 首尾都是 1。
  • 第 2 行：先放首尾的 1，中间位置 (2,1) = 上一行的 (1,0) + (1,1) = $1 + 1 = 2$ → [1, 2, 1]。
  • 第 3 行：首尾为 1，中间 (3,1) = (2,0) + (2,1) = 1 + 2 = 3，(3,2) = (2,1) + (2,2) = 2 + 1 = 3 → [1, 3, 3, 1]。

依此类推。

这个过程的本质就是动态规划中的”填表”——每一行的值只依赖上一行的值，没有后效性。我们只需要保存上一行的结果，就可以推导出当前行。

到这里，我们可以问自己一句：还有比这更优的解法吗？

组合数公式需要 $O(n^3)$ 甚至 $O(n^2)$（带预处理），而逐行构建法每个位置只做一次加法，已经达到了理论上的下界——因为输出本身就包含 $\frac{n(n+1)}{2}$ 个数字，至少要 $O(n^2)$ 时间才能生成出来。

所以这就是最优解了。

复杂度分析

版本 时间复杂度 空间复杂度 说明
暴力法（组合数公式） $O(n^3)$ $O(n^2)$ 每计算一个组合数都算阶乘，效率低
优化组合数（预处理阶乘） $O(n^2)$ $O(n^2)$ 预计算阶乘和逆元，但已不必要
逐行构建（动态规划） $O(n^2)$ $O(n^2)$ 每个格子一次加法，最优

时间复杂度 $O(n^2)$ 很好理解：我们要生成大约 $n^2/2$ 个数字，每个数字是常数时间的加法。

空间复杂度 $O(n^2)$ 来自于存储整个三角所需的空间。如果只看额外空间，我们实际上只保留了上一行（$O(n)$），但题目要求返回全部结果，所以输出空间本身已经是 $O(n^2)$ 了。

延伸思考

这道题让我们学到了什么？

定义本身就是解法的最优切入点。 很多时候，数学定义（组合数公式）虽然严格，但未必是最适合编程实现的。反而是定义背后那个直观的性质（每个数字等于上方两数之和）给了我们最优的算法。

这其实是一种常见的思维转变：从”公式计算”转向”递推推导”。在杨辉三角里表现为逐行构建，在其他问题里可能表现为动态规划、递推关系或者迭代更新。

下次遇到类似问题优先观察什么？

  1. 有没有递推关系？ 当前状态能否通过相邻的上一步状态简单推导出来？这就是动态规划的雏形。
  2. 边界条件是否特殊？ 杨辉三角每行的首尾是 1，很多 DP 问题都有类似的”地基”需要先打好。
  3. 输出规模是多少？ 如果能提前意识到输出本身的规模是 $O(n^2)$，我们就不会指望找到一个 $O(n)$ 的算法，从而更安心地接受当前解。

一句话总结：杨辉三角的生成就是不断问自己”上一行告诉我什么”，然后把这个答案向前推进一步。递归地看是递推，迭代地看就是逐行构建——大道至简。
