盛最多水的容器

盛最多水的容器

核心思路

第一步：读懂问题

给定一个长度为 n 的整数数组 height，每个元素代表一条垂线的高度。我们要找出其中的两条线，使得它们与 x 轴共同构成的容器能容纳最多的水。

换句话说，选两个下标 i 和 j（i < j），容器的容量由两个因素决定：

  • 宽度：两条线之间的距离 j - i
  • 高度：由两条线中较短的那条决定（水会从短板溢出）

我们的目标就是找到 max(area)。

第二步：暴力法 —— 枚举所有可能

如果没有任何技巧，我们第一反应会怎么做？

不妨想一下，最直接的方式就是枚举所有可能的线对。两层循环，外层固定左线，内层遍历右线，计算每个容器面积并更新最大值。

这个解法的问题在哪里呢？

当 n = 10^5 时，n(n-1)/2 ≈ 5 × 10⁹ 次计算，显然会超时。时间复杂度是 O(n²)，而空间复杂度是 O(1)。

那么，有没有办法减少不必要的枚举呢？

第三步：寻找优化方向

我们来观察面积公式：

面积由短板高度和宽度共同决定。

假如我们一开始就把两个指针放在最左和最右，此时宽度最大。然后我们移动指针，宽度必然减小。为了让面积变大，唯一的希望就是让短板变高。

这就引出了一个关键的自问：

当我们固定了两条线中的一条，移动另一条时，面积会如何变化？

让我们用一个具体的例子来感受一下。

假设 height = [1, 8, 6, 2, 5, 4, 8, 3, 7]，左右指针分别指向 1 和 7。

此时容器高度取决于短板 1，面积是 1 × 8 = 8。

现在我们面临两个选择：

  1. 移动较高的线（右边）：新的容器高度最多是 1（因为短板还是左边这个 1），宽度却变小了，面积只可能变小或不变。
  2. 移动较矮的线（左边）：短板可能变成一个新的更高的值，面积有可能变大。

所以直觉告诉我们：每次移动较矮的那条线。

第四步：证明正确性

为什么要移动较矮的那条？我们来严格地想一想。

设左右指针为 l 和 r，且 height[l] < height[r]。当前面积是 height[l] * (r - l)。

如果我们固定 l 向左移动 r，那么对于任意 l' = l、r' < r：

  • 宽度 r' - l < r - l，变小了
  • 高度 min(height[l], height[r']) ≤ height[l]（因为 height[l] 已经是短板了，换一个右边只会更矮或者不变）

所以面积一定小于 height[l] * (r - l)。换句话说，对于当前的左线 l 来说，右线是 r 时已经得到了它能参与的最大面积。继续用 l 和任何其他右线配对都不会更大，所以可以直接抛弃 l，左指针右移。

反之如果 height[r] 是短板，同理，抛弃 r，右指针左移。

这个证明告诉我们：每次移动短板指针是安全的，不会错过最优解。

第五步：双指针法的完整过程

  1. 初始化左指针 l = 0，右指针 r = n - 1
  2. 计算当前面积，更新最大值
  3. 比较 height[l] 和 height[r]，移动较短的指针
  4. 重复直到 l >= r

这个过程每一步都在缩小搜索范围，但保证不会丢掉最优解。它本质上是一种贪心策略——每次都做出局部最优的选择（舍弃不可能成为答案的线对）。

延伸思考

这道题让我们学到了什么？

双指针缩减搜索空间 + 贪心决策。我们并不是盲目地双指针，而是找到了一个单调性——当固定短板时，移动长板面积只会变小，因此可以安全地抛弃短板。这个思路的本质是：每一步都找到当前局面下”不可能产生更优解”的候选，将其排除。

这种证明方式在双指针问题中非常常见（比如 “三数之和” 也用到了类似的思想——排序后根据和的大小决定移动哪个指针）。

下次遇到类似问题优先观察什么？

  • 能否用两个指针从两端向中间移动？ 通常当问题的答案涉及两个元素之间的距离时，可以考虑双指针。
  • 有没有单调性可以利用？ 当我们固定一个变量，另一个变量的变化是否导致结果单调变化？找到这个单调性就可以大胆地移动指针。
  • “短板效应”的直觉：如果结果受限于某个”短板”，尝试移动短板去寻找更优解。

最后留一个小问题：如果容器可以倾斜（即水可以没过较高的线），那解法会有什么不同呢？感兴趣的话可以搜索「Trapping Rain Water」——也就是接雨水问题，它在盛水问题上又进了一步。
