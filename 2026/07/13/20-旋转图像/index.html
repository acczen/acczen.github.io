LeetCode 48. 旋转图像（rotate-image）

题目

LeetCode 48. rotate-image

给定一个 $n \times n$ 的二维矩阵 matrix 表示一个图像。请你将图像顺时针旋转 90 度。

你必须在原地旋转图像，这意味着你需要直接修改输入的二维矩阵。请不要使用另一个矩阵来旋转图像。

提示：

  • $n = matrix.length = matrix[i].length$
  • $1 \le n \le 20$
  • $-1000 \le matrix[i][j] \le 1000$

核心思路

拿到这道题，我们先看看能不能”暴力”解决。

找到目标位置

最直接的想法：我们创建一个同样大小的新矩阵 newMatrix，然后对于原矩阵的每个元素 matrix[i][j]，算出它在旋转后的新位置，填进去，最后拷贝回来。

现在的问题是：matrix[i][j] 旋转 90 度后去哪了？

让我们用 $3 \times 3$ 的例子找规律。把坐标标出来：

你发现了什么？旋转后：

  • 原来的行号变成了新位置的列号（而且是倒序）
  • 原来的列号变成了新位置的行号

更精确地说，matrix[i][j] 旋转后的位置是 newMatrix[j][n-1-i]。验证一下：左上角 (0,0) 去了 (0, 2)，即 newMatrix[0][3-1-0] = [0][2]，对应输出第一行第三列的 7 … 不对，(0,0) 的值是 1，旋转后应该在最右上角也就是第一行第三列——没错，输出是 [7,4,1]，1 确实在第 0 行第 2 列。

所以借助额外数组的解法是可行的，时间 $O(n^2)$，空间 $O(n^2)$。

但题目明确要求原地旋转。难点就在这——我们不能让所有元素同时搬家，因为每次修改都会覆盖掉还没用过的旧值。怎么办？

能不能”逐个换位”？

一个自然的想法是：像拧魔方一样，看一个元素旋转 4 次之后回到原位。比如 (0,0) 上的值 1，旋转 90 度到 (0,2)，再旋转 90 度到 (2,2)，再到 (2,0)，最后回到 (0,0)。这恰好是一个四元环！

如果能把每个这样的”四人组”一次性轮换到位，就能只用一个临时变量完成原地旋转。这是一种可行的思路（称”环形替换”），但需要处理”哪些位置是新的轮换起点”这个边界问题。

不过，还有没有更直观、更不容易写错的方法？矩阵操作中，有没有”用已知操作组合出新操作“的技巧？

换个角度：把旋转拆成两步

回想一下：在”轮转数组”那道题里，我们用”反转”这个简单操作，通过三次组合完成了轮转。矩阵有没有类似的”原子操作”可以用？

顺时针旋转 90 度，可以拆成两个更简单、各自也容易在原地实现的操作：

  1. 转置（Transpose）：沿着主对角线翻转，matrix[i][j] ↔ matrix[j][i]
  2. 水平翻转（每行反转）：把每一行的元素左右颠倒

来看看对不对：

与示例输出 [[7,4,1],[8,5,2],[9,6,3]] 完全一致！

这样一来，问题就变成了”如何原地完成转置和水平翻转”——两个小问题，各自都非常清晰。

转置只需要遍历主对角线右上方的元素（即 $j > i$ 的元素），与对称位置的元素交换。水平翻转更简单——对每一行用双指针交换首尾元素直到中间。

另一种拆法（逆时针旋转 90 度则是”转置 + 垂直翻转”），但顺时针这道题用”转置 + 水平翻转”最自然。

总结思考链条

暴力法（借助额外矩阵观察规律）→ 发现四元环替换 → 但实现复杂 → 想到”用简单操作组合出旋转” → 转置 + 行翻转，两步都在 $O(1)$ 空间内轻松完成。

解法：转置 + 水平翻转

为什么选择这个解法？

方案 时间 空间 代码复杂度
额外矩阵 $O(n^2)$ $O(n^2)$ 低，但不满足原地要求
四元环替换 $O(n^2)$ $O(1)$ 高，需处理边界和循环起点
转置 + 水平翻转 $O(n^2)$ $O(1)$ 低，两个简单操作的组合

转置 + 水平翻转的妙处在于：每一步的含义都非常明确，各自可以用经典的双指针/对称交换实现，几乎不会写出 bug。面试中这个解法也容易解释清楚——“顺时针旋转 = 转置 + 水平翻转”是一个经典的线性代数结论。

算法步骤

  1. 转置：遍历矩阵的上三角区域（$j > i$），交换 matrix[i][j] 和 matrix[j][i]。
  2. 水平翻转：对于每一行，反转该行所有元素。即交换 matrix[i][j] 和 matrix[i][n-1-j]，$j$ 从 $0$ 到 $\lfloor n/2 \rfloor - 1$。

两步可以分开写，也可以合并为同一个双层循环（但分开写更清晰，减少出错的可能）。

复杂度分析

维度 分析
时间复杂度 $O(n^2)$ — 转置遍历了约 $n(n-1)/2$ 个元素，水平翻转遍历了全部 $n^2$ 个元素（每行 $n/2$ 次交换，共 $n$ 行），总体 $O(n^2)$。矩阵中的每个元素至少被访问一次是绕不开的下限——因为旋转操作改变了每个元素的依赖关系。
空间复杂度 $O(1)$ — 只用了几个临时变量做交换，没有任何与 $n$ 相关的额外空间分配。

延伸思考

学到了什么？

矩阵的复合变换（旋转）往往可以分解为若干简单原子操作的组合——就像轮转数组里的”反转”一样，转置和行/列翻转是矩阵操作中最基础也最实用的原子动作。熟悉这些简单操作，以后遇到更复杂的矩阵变换时，就能试着问自己：能不能把它拆成转置 + 翻转 + 行交换之类的组合？

下次遇到类似问题，你会优先观察什么？

遇到要求”原地变换矩阵/数组”的题目，优先思考：这个变换能否分解为两个或多个你已经会写的简单操作？ 拿出纸笔，先画一个 $3 \times 3$ 的小例子，跟踪每个元素在分解步骤中的位置变化——很多时候，结论会比你预想的更优雅。
