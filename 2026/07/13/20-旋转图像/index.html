核心思路

第一反应：新矩阵拷贝

题目要求我们将一个 n×n 的二维矩阵顺时针旋转 90 度。不妨想一下，如果没有任何技巧，我们第一反应会怎么做？

最简单直白的做法就是申请一个同样大小的新矩阵，然后遍历原矩阵，把每个元素放到它旋转之后应该去的位置。

对于一个 n×n 矩阵，旋转 90 度后，位置 (i, j) 上的元素会去到 (j, n - 1 - i)。这一点我们可以用一个简单的例子验证：左上角 (0, 0) 会去到 (0, n-1)，也就是右上角；右上角 (0, n-1) 会去到 (n-1, n-1)，也就是右下角——刚好对应顺时针旋转一圈的轨迹。

这个解法的时间复杂度是 O(n²)，因为我们遍历了所有 n² 个元素。空间复杂度同样是 O(n²)，因为我们额外申请了一个同样大小的矩阵。

问题出在哪里？ 题目要求原地修改，不允许额外分配矩阵。我们需要想一个不需要额外空间的方案。

能不能省下一行数组的空间？

既然不能直接开一个新矩阵，我们可能会想：至少能不能省一点空间？比如只用一个一维数组来暂存一行？

旋转一行的操作其实是有规律的：顶行的元素会跑到右列，右列跑到底行，底行跑到左列，左列跑到顶行。如果我们用一个临时数组保存顶行，然后依次把左列赋给顶行、底列赋给左列……这样确实只需要 O(n) 的额外空间。

但问题是，这种方式操作起来比较繁琐，而且我们仍然没有达到原地操作的要求——“原地”意味着除了常数个变量外，不能使用任何额外空间。

进一步思考：如何做到真正的原地？

有没有办法让每个元素直接覆盖到它该去的位置，而不需要额外存储一整行呢？

有意思！四个角上的元素刚好形成一个长度为 4 的循环。不止是角上的元素，实际上，矩阵中每四个对称位置上的元素都构成一个长度为 4 的循环。

那么我们可以这样操作：对于每个这样的四元组，用一个临时变量保存第一个元素，然后依次把后一个元素覆盖到前一个位置，最后把临时变量赋给最后一个位置。这样一来，我们只需要 O(1) 的额外空间。

哪些位置需要参与这样的交换？我们只需要处理矩阵的四分之一部分（比如左上角那块）即可——因为每个四元组覆盖了四个不同象限的位置。

对于 n×n 矩阵，我们按层处理。最外层从 i = 0 遍历到 n/2 - 1（层数），内层从 j = i 遍历到 n - i - 2（避免重复处理最后一个元素，因为它是一个四元组的起点也是上一个的终点）。

四个位置的关系是：

  • 顶：(i, j)
  • 右：(j, n - 1 - i)
  • 底：(n - 1 - i, n - 1 - j)
  • 左：(n - 1 - j, i)

这样我们就得到了一个空间 O(1) 的原地旋转算法。

更优雅的思路：转置 + 翻转

不过说实话，上面那种逐层四元组交换的方法虽然做到了原地，但下标关系有点绕，写起来容易出错。有没有更直观的方式？

我们不妨换个角度思考：顺时针旋转 90 度，能不能分解成两个更简单的操作？

试着想想——如果我们先把矩阵沿主对角线翻转（即转置），然后每一行水平翻转，会得到什么？

反过来，如果先水平翻转再转置，得到的是逆时针旋转 90 度。

这个方法最大的优点就是思路清晰：我们不需要记忆复杂的四元组下标关系，只需要理解”转置”和”翻转”两个直观的操作，组合起来就能达到目标。而且两个步骤各自都非常容易实现。

复杂度分析

版本 时间复杂度 空间复杂度 说明
暴力法（新矩阵拷贝） O(n²) O(n²) 遍历所有 n² 个元素，额外分配 n×n 矩阵
四元组原地交换 O(n²) O(1) 按层遍历四分之一矩阵，每四个元素一组交换
转置 + 翻转 O(n²) O(1) 先转置再逐行反转，代码最简洁

三种方法的时间复杂度都是 O(n²)，因为无论如何都需要处理矩阵中的每一个元素。但空间上，转置+翻转和四元组交换都达到了 O(1) 的原地操作。

延伸思考

这道题让我们学到了什么？

把一个复杂的变换拆解成多个简单操作的组合——这种思路远比硬记下标关系更有价值。旋转 90 度不好直接写，但转置和水平翻转都是我们非常熟悉的基本操作，把它们组合起来，问题就迎刃而解。很多看似复杂的矩阵或数组变换问题，都可以通过这种方式拆解。

另外，几何变换中的”四元组循环”也是一个值得留意的模式——当我们需要在原地完成某种对称变换时，找到元素之间形成的置换环，每次处理一个环上的所有元素，就能以 O(1) 额外空间完成任务。

下次遇到类似问题优先观察什么？

  • 如果问题是关于矩阵变换（旋转、翻转、对角线对称等），不妨先考虑是否能拆解成已知的基本操作
  • 尝试追踪一个元素在变换前后的位置变化，看看有没有形成规律性的循环
  • 对于 n×n 方阵，转置操作是一个特别有用的中间步骤，值得牢记
  • 如果题目要求原地操作，要关注额外空间的使用——常数空间往往意味着我们需要利用元素本身的存储位置来完成”搬运”
