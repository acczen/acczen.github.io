N 皇后

N 皇后

核心思路

1. 问题在问什么？

题目要求我们把 N 个皇后放到 N×N 的棋盘上，使得任意两个皇后不互相攻击。在国际象棋里，皇后可以横着走、竖着走、斜着走，距离不限。换句话说，任意两个皇后不能在同一行、同一列、同一条对角线。

我们不妨从最简单的规模入手：N = 4 时，棋盘 4×4，放 4 个皇后，有解吗？有几个解？

2. 最直接的想法

如果没有任何限制，我们要从 4×4 = 16 个格子里挑 4 个放皇后，那就是组合数 C(16, 4) = 1820 种可能。对每一种我们都要检查是否冲突——N 稍大一点，这个数字就会爆炸式增长。

问题来了：有没有办法减少搜索量？

稍微想一下就能发现：每行只能放一个皇后。因为如果有两个皇后在同一行，它们一定能互相攻击。所以我们可以逐行放置，每行只选一列。

这样，N = 4 时搜索空间从 C(16,4) 降到了 4⁴ = 256 种。好了一些，但还是很大。

3. 还能再剪吗？

既然每行只能放一个，那每列是不是也只能放一个？是的！所以当我们决定第 i 行的皇后放在第 j 列时，这一列就不能再放其他皇后了。

等等——这听起来像是排列问题：第 0 行选一个列、第 1 行选一个不同的列、第 2 行选一个还没用过的列……不就是 N 个列的全排列吗？

N = 4 时，4! = 24 种排列。比起最初的 1820 已经大幅缩小了。但是，排列只是保证了不同行不同列，对角线约束还没有处理。

4. 对角线怎么判断？

我们不妨画一个 4×4 棋盘，把每个格子标上 (row, col)：

主对角线（从左上到右下）上的格子有什么共同点？观察：

  • (0,0)、(1,1)、(2,2)、(3,3) → row - col = 0
  • (0,1)、(1,2)、(2,3) → row - col = -1
  • (1,0)、(2,1)、(3,2) → row - col = 1

规律：同一条主对角线上，row - col 是常数。

副对角线（从右上到左下）呢？

  • (0,3)、(1,2)、(2,1)、(3,0) → row + col = 3
  • (0,2)、(1,1)、(2,0) → row + col = 2
  • (1,3)、(2,2)、(3,1) → row + col = 4

规律：同一条副对角线上，row + col 是常数。

有了这两个恒等式，检查对角线冲突就变成了 O(1) 的查表操作！

5. 完整算法：回溯 + 剪枝

我们使用回溯（Backtracking）框架：

  1. 从第 0 行开始，逐行尝试放置皇后
  2. 对于当前行，遍历每一列，检查三个条件：
    • 该列没有被占用（用 cols 布尔数组记录）
    • 该主对角线没有被占用（用 diag1 布尔数组记录，下标为 row - col + N - 1，防止负数）
    • 该副对角线没有被占用（用 diag2 布尔数组记录，下标为 row + col）
  3. 如果都满足，就在当前位置放置皇后，标记三个数组，递归到下一行
  4. 递归返回后撤销标记（回溯）
  5. 当 row == N 时，说明全部放完，记录当前棋盘布局

6. 为什么这是最优解？

  • 每行只放一个，天然满足行约束
  • cols 数组保证列不重复
  • diag1 和 diag2 数组保证两条对角线不重复
  • 每一步只尝试可行的列，一旦发现冲突立即剪枝，不需要等到最后才验证

这个框架对 N 皇后问题的所有变体（比如只求数量而不是具体解）都适用，只需调整记录方式即可。

复杂度分析

版本 时间复杂度 空间复杂度 说明
暴力枚举 C(N², N) $O(C(N^2, N) \cdot N^2)$ $O(N^2)$ 不可行，N=8 时就无法承受
逐行枚举 N^N $O(N^N \cdot N)$ $O(N)$ 仍然指数爆炸
排列搜索 N! $O(N! \cdot N)$ $O(N)$ 比前两种好很多，但没处理对角线
回溯 + 三重剪枝（最优解） $O(N!)$ $O(N)$ 实际远小于 N!，因为对角线剪枝在早期就能砍掉大量分支

详细说明：

  • 时间复杂度： 最坏情况下上界是 $O(N!)$，因为每一行尝试的列数递减（第一行 N 种可能，第二行最多 N-1 种……）。但由于对角线约束的存在，实际搜索空间远小于 N!。例如 N=8 时 8! = 40320，但实际回溯只需检查约 15720 个节点。
  • 空间复杂度： $O(N)$，queens 数组长度 N，三个布尔数组长度分别为 N、2N-1、2N-1，递归调用栈深度 N。

延伸思考

这道题让我们学到了什么？

  1. 逐层约束比全局检查更高效。 如果我们在最后一刻才验证所有皇后是否冲突，浪费了大量无意义的搜索。把约束拆解到每一步，能尽早剪枝。

  2. 找对问题的数学结构是关键。 行、列、对角线的冲突判断都有对应的数学表示——行用索引、列用布尔数组、对角线用 row ± col 常数。找对了表示方式，判断就从 $O(N)$ 降到了 $O(1)$。

  3. 回溯的三段式是模板：做选择 → 递归 → 撤销选择。 几乎所有”在约束下枚举所有排列/组合”的问题都可以套用这个框架。

下次遇到类似问题优先观察什么？

  • 搜索空间的维度是什么？ 是逐行、逐列，还是逐格？选择正确的枚举顺序能大幅降低分支因子。
  • 约束条件能否独立追踪？ 对于”两两不能冲突”类的问题，想一想每一条约束是否有简洁的数学表示（数组、哈希集合、位运算掩码）。
  • 局部冲突与全局验证的权衡。 每一步判断冲突的代价有多大？如果判断很贵（比如需要扫描整个棋盘），也许需要换一个更高效的数据结构。

N 皇后是回溯搜索的经典入门题，但它背后的思想——把”全局合法性验证”拆解为”局部约束的快速判断”——几乎贯穿所有组合搜索问题。理解了它，排列、组合、子集、数独、括号生成这些问题都会变得清晰许多。
