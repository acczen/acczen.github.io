二叉树的直径

二叉树的直径

核心思路

第一步：理解题目在问什么

题目给出一棵二叉树，让我们找出它的”直径”——也就是树中任意两个节点之间最长路径的长度。这里的路径长度用边数来度量。

让我们先看一个简单的例子：

这棵树中最远的两个节点是 4 和 5（或者 4 和 3，路径长度也一样）。节点 4 到节点 5 的路径是 4 → 2 → 5，经过了 2 条边，所以直径是 2（注意不是 3，虽然路径上走了 3 个节点，但边数是 2）。

再来看一个情况：

这里最远的两个节点是 5 和 4，路径是 5 → 3 → 2 → 4，边数为 3。注意这条路径并没有经过根节点 1。这说明直径不一定经过根，我们必须在整棵树的所有节点上都考虑可能性。

第二步：从暴力解法开始思考

不妨想一下：如果我们要暴力求解，该怎么做？

最直接的想法：枚举所有节点对，计算每对之间的距离，取最大值。但是树中有 N 个节点，节点对就有 O(N^2) 个，计算每对的距离还需要 O(N) 的时间，这显然不可接受。

让我们换个角度思考：一条路径长什么样？

对于树中的任意一条路径，它一定有一个”最高点”——也就是路径上深度最小的那个节点。如果我们把这个节点叫做 node，那么这条路径就是从 node 的左子树中某个节点出发，向上走到 node，再向下走到右子树中的某个节点。

换句话说，以 node 为最高点的路径长度 = node 左子树的最大深度 + node 右子树的最大深度。

这里的”深度”是指从 node 到最远叶子节点的边数。

所以直径就是：对于所有节点，计算”左子树最大深度 + 右子树最大深度”，取最大值。

第三步：深度优先搜索（DFS）

想清楚了这个关系，解法就呼之欲出了——我们只需要一次 DFS 遍历，在计算每个节点深度的同时，顺手更新直径的最大值。

对于每个节点 node：

  • 递归计算左子树的最大深度 leftDepth
  • 递归计算右子树的最大深度 rightDepth
  • 经过 node 的最长路径（即以 node 为最高点的路径）长度为 leftDepth + rightDepth，用它更新全局答案
  • 返回 max(leftDepth, rightDepth) + 1 作为 node 的深度，供父节点使用

让我们用手上的例子走一遍：

  • 从节点 4（空左右子树）开始：left=0, right=0, 直径候选=0, 返回深度=1
  • 节点 5 同理：left=0, right=0, 直径候选=0, 返回深度=1
  • 节点 2：left=1（来自4）, right=1（来自5）, 直径候选=1+1=2, 更新全局最大=2, 返回深度=max(1,1)+1=2
  • 节点 3：left=0, right=0, 直径候选=0, 返回深度=1
  • 节点 1：left=2（来自2）, right=1（来自3）, 直径候选=2+1=3, 更新全局最大=3…等等！

这里节点 1 算出来直径候选是 3，但实际这棵树的直径是 2（4 到 5）啊，哪来的 3？

这里有一个非常关键的细节：我们之前说过直径的度量单位是边数。节点 1 的 left=2 是什么意思？节点 1 到节点 4 的路径是 1 → 2 → 4，边数是 2。同样，节点 1 到节点 3 的路径是 1 → 3，边数是 1。所以 left+right = 3 对应了路径 4 → 2 → 1 → 3，正是 3 条边。

这棵树的最长路径确实是 3 条边（4 → 2 → 1 → 3 或者 5 → 2 → 1 → 3），而不是 2。我之前看错了——4 到 5 的路径长度是 2，但 4 到 3 的路径长度是 3，所以直径应该是 3 才对。

验证一下：LeetCode 官方示例中，这棵树的直径确实是 3。所以我们的思路是对的。

第四步：为什么这个解法是正确的？

让我们再推敲一下：有没有可能最长路径的最高点不是某个节点，而是某条边？

不会。在树中，任意一条路径都必然有唯一的一个”最高点”（即路径上距离根最近的那个节点）。这是因为树是无环的连通图，路径的两个端点之间有且仅有一条简单路径，而这条路径上深度最小的节点就是最高点。

所以，只要我们枚举了所有节点作为最高点，就枚举了所有可能的路径，取最大值就是直径。

第五步：时间复杂度分析

每个节点在递归中恰好访问一次，所以时间复杂度是 O(N)。空间复杂度方面，递归深度在最坏情况下（树退化成链表）可以达到 N，所以是 O(N)，平均情况下（平衡树）是 O(log N)。

延伸思考

这道题让我们学到了什么？

将全局最优解拆解为局部最优解的组合。 二叉树的直径问题告诉我们：树上的路径问题，往往可以固定路径上的”最高点”，然后问题就转化为求子树深度——一个我们很熟悉的问题。

这种”枚举最高点”的思路，在树形 DP 中非常常见。很多树上的路径问题（比如”二叉树中的最大路径和”——LeetCode 124）都沿用了这个框架：后序遍历，在归的过程中整合左右子树的信息，更新全局答案。

下次遇到类似问题优先观察什么？

  1. 路径的形态：路径是一笔画还是可以分叉？经过一个节点时，是只能往下走（单向）还是可以左右汇聚（双向）？本题中路径可以左右汇聚，所以用 left + right。

  2. 衡量单位：是边数还是节点数？本题是边数，所以空节点返回 0，单节点深度为 0。如果要求的是节点数，空节点返回 0，单节点深度需要返回 1。

  3. 答案的位置：是在递归过程中维护全局变量，还是返回值本身就包含答案？本题用全局变量更自然，因为最终的直径不一定在根节点处。

  4. 树的平衡性：如果题目暗示树很大或者可能退化成链表，可以提前想好递归是否会导致栈溢出，必要时改为迭代写法。

下次拿到一道树上的路径最值问题，不妨先问自己三个问题：路径能不能拐弯？单位是边还是点？答案一定在根节点吗？ 想清楚这几个问题，解法就有了方向。
