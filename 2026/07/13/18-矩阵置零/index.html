LeetCode 73. 矩阵置零（set-matrix-zeroes）

题目

LeetCode 73. set-matrix-zeroes

给定一个 $m \times n$ 的矩阵，如果一个元素为 $0$，则将其所在行和列的所有元素都设为 $0$。请使用原地算法。

示例 2：

提示：

  • $m == matrix.length$
  • $n == matrix[0].length$
  • $1 \leq m, n \leq 200$
  • $-2^{31} \leq matrix[i][j] \leq 2^{31} - 1$

进阶： 一个直观的解决方案是使用 $O(mn)$ 的额外空间，但这并不是一个好的解决方案。一个简单的改进方案是使用 $O(m+n)$ 的额外空间，但这仍然不是最好的解决方案。你能想出一个仅使用常量空间的解决方案吗？

核心思路

拿到这道题，我们先别急着写代码。让我们像侦探一样，从最笨的办法开始，一步步挖出那个”啊哈！”时刻。

第一层：暴力复制

最直接的想法是什么？遍历一遍矩阵，把所有的 $0$ 记下来，然后再遍历一遍，把对应的行和列都置零。

这需要多少空间？我们需要记住每一个 $0$ 的位置，最坏情况下整个矩阵都是 $0$，那就需要 $O(mn)$ 的空间。

为什么这个不够好？ 题目明示了：$O(mn)$ 的额外空间”并不是一个好的解决方案”。我们可以做得更省。

第二层：行列标记数组

注意一个关键事实：我们关心的是哪些行有 $0$、哪些列有 $0$，而不是每个 $0$ 的精确坐标。 一行有多个 $0$，和我们只需知道这一行”存在 $0$”——标一次就够了。

这就引出了 $O(m+n)$ 的方案：用一个长度为 $m$ 的布尔数组标记哪些行要清零，用一个长度为 $n$ 的布尔数组标记哪些列要清零。第一遍扫描时打标记，第二遍根据标记执行清零。

空间从 $O(mn)$ 降到了 $O(m+n)$。但题目还在追问：能不能只用常量空间？

第三层：把标记藏进矩阵本身

$O(1)$ 意味着我们不能额外开辟和输入规模相关的数组。那我们得把标记信息存在哪里呢？

问问自己：矩阵本身有 $m \times n$ 个格子，我们能不能”借用”其中一些格子来当标记？

这就引出了一个漂亮的想法：用矩阵的第一行和第一列来当我们的标记数组。

具体来说：

  • matrix[i][0] 为 $0$，表示第 $i$ 行需要清零（$i > 0$）
  • matrix[0][j] 为 $0$，表示第 $j$ 列需要清零（$j > 0$）

这样就省掉了那两个额外的布尔数组——我们把标记直接写在矩阵的第一行和第一列里。

但这里有一个陷阱：那第一行和第一列自己的”命运”呢？如果第一行原本就有 $0$，它自己也需要被清零，但我们用它的格子来存别人的标记，改了它之后，我们怎么知道它原本是否包含 $0$？

这个问题你可以暂停想一想……答案其实很直接：用两个单独的布尔变量，提前记录第一行和第一列原本是否包含 $0$。 因为只有这两个维度的信息需要额外保存，而且它们是常量——不管矩阵多大，都是两个变量。

思考链条回顾

  1. 暴力记录所有 $0$ → $O(mn)$ 空间，冗余
  2. 发现我们只需知道”哪些行、哪些列” → $O(m+n)$ 空间
  3. 发现可以从矩阵本身”借用”第一行和第一列来当标记 → $O(1)$ 空间，但需要两个变量保护首行首列

这就是从 $O(mn)$ 到 $O(m+n)$ 再到 $O(1)$ 的完整思考过程。每一步的精髓都不同：第一步在去除冗余，第二步在借用已有空间。

解法：原地标记法

现在我们把这个思路落实到具体步骤上。

为什么选择这个方案？

我们对比一下备选方案：

方案 空间 优点 缺点
复制整个矩阵 $O(mn)$ 思路简单，不会覆盖原始数据 空间浪费巨大
行列标记数组 $O(m+n)$ 去掉了坐标冗余 仍需要线性额外空间
首行首列标记 $O(1)$ 完全原地，空间最优 需要小心处理首行首列

显然，第三个方案是本题的”正解”——它完美利用了矩阵自身的空间，把额外开销降到常数级。

算法步骤

设矩阵为 $matrix$，有 $m$ 行 $n$ 列。

第一步：检查第一行和第一列。

我们需要先知道第一行和第一列原本是否包含 $0$。为什么必须”先”？因为后面我们会用第一行和第一列的格子来做标记，它们的值会被修改，到时候就分不清是”原本的 $0$”还是”后来写的标记”了。

第二步：遍历内部区域，用第一行和第一列打标记。

从 $(1,1)$ 开始遍历矩阵的”内部”（去掉第一行和第一列）。每当 matrix[i][j] == 0，就把这一行和这一列的”标志位”设为 $0$：

  • matrix[i][0] = 0 → “第 $i$ 行有 $0$”
  • matrix[0][j] = 0 → “第 $j$ 列有 $0$”

比如，我们发现在 $(2,3)$ 有一个 $0$，就把 matrix[2][0] 和 matrix[0][3] 都设为 $0$。

第三步：根据标记清零内部区域。

再次遍历内部区域，检查每一格对应的”行标志”或”列标志”是否为 $0$：

  • 如果 matrix[i][0] == 0 或 matrix[0][j] == 0，就把 matrix[i][j] 设为 $0$。

注意：这里检查的条件是或——行或列只要有一个被标记，这个格子就得清零。

第四步：根据预先记录的两个变量，处理第一行和第一列。

  • 如果 firstRowHasZero 为真，把整个第一行清零。
  • 如果 firstColHasZero 为真，把整个第一列清零。

为什么不能打乱顺序？

你可能会想：能不能在第二步就顺便把首行首列也标记了？问题在于，如果我们用 matrix[0][0] 同时表示”第一行有 $0$”和”第一列有 $0$”，这两个信息会互相覆盖——一个格子存不下两个含义。

这就是为什么我们单独用两个变量来记住首行首列的状态，而 matrix[0][0] 只用来参与内部区域的标记（它同时是”第一行标记数组”的第一个元素和”第一列标记数组”的第一个元素——但这两个标记数组的交集就是它本身，所以会冲突。把首行首列用独立变量记录，就完美解决了这个冲突）。

复杂度分析

维度 分析
时间复杂度 $O(m \times n)$ — 我们遍历了矩阵常数次（检查首行首列一次，打标记一次，清零一次，处理首行首列一次），每次都是 $O(mn)$，常数倍不影响复杂度
空间复杂度 $O(1)$ — 只用了两个布尔变量，与输入规模无关

延伸思考

学到了什么？

“借用已有空间”是一种强大的思维模式。 当题目要求 $O(1)$ 空间但你又需要额外的存储时，不妨环顾四周——输入数组本身能不能”兼职”做你的辅助结构？本题用第一行第一列做标记数组、原地哈希中用数组索引存信息、甚至链表反转中复用节点指针，都是同一套心法。

下次遇到类似问题，你会优先观察什么？

先问自己：“我需要记住的信息，能不能写在输入数据本身里面，而不是另开一个新的数组？” 如果可以，下一个问题就是——“我写进去之后，怎么把它和原始数据区分开？” 本题的答案是用两个独立变量保护首行首列；别的题可能用正负号、位移、甚至不同的数值范围来区分。核心矛盾永远是：如何在不丢失原始信息的前提下，复用现有空间。
