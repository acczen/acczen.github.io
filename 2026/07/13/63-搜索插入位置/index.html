搜索插入位置

搜索插入位置

核心思路

题目： 给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。要求时间复杂度为 O(log n)。

第一反应：暴力扫描

拿到这道题，最简单的想法就是——遍历数组。

从左到右扫描一遍：

  • 如果遇到 nums[i] == target，直接返回 i；
  • 如果遇到 nums[i] > target，说明 target 应该插在 i 的位置，返回 i；
  • 如果遍历完都没找到，说明 target 比所有元素都大，返回 n（数组长度）。

这个解法正确吗？正确。简洁吗？很简洁。但问题是——时间复杂度是 O(n)。

题目要求 O(log n)，这意味着我们需要换个思路。

为什么能更快？——抓住关键条件

让我们停下来想一想：题目给了我们一个什么特殊条件？

数组是排序的。

排序，意味着什么？意味着我们可以利用元素之间的顺序关系来缩小搜索范围，而不必逐个检查。

想象一下查字典：要找 “binary” 这个词，你会从第一页翻到最后一页吗？不会。你会直接翻到字典中间，看看 “b” 在中间页的前面还是后面，然后继续折半查找。

这就是二分查找的思想。

二分查找的变体

标准的二分查找用来找 target 是否存在。但这道题多了一个要求：如果不存在，返回插入位置。

不妨想一下，二分查找结束时，我们的搜索区间会缩到一个点上。这个点，恰好就是 target 应该出现的位置。

我们用两个指针 left 和 right 表示当前的搜索区间 [left, right]（注意是闭区间）。每次取中间位置 mid：

  • 如果 nums[mid] == target：找到了，直接返回 mid。
  • 如果 nums[mid] < target：target 在右半部分，缩小到 [mid + 1, right]。
  • 如果 nums[mid] > target：target 在左半部分，缩小到 [left, mid - 1]。

当 left > right 时循环结束。此时 left 指向的是什么？

让我们用一个例子走一遍：nums = [1, 3, 5, 6], target = 2。

  1. left = 0, right = 3, mid = 1, nums[1] = 3 > 2 → right = 0
  2. left = 0, right = 0, mid = 0, nums[0] = 1 < 2 → left = 1
  3. left = 1, right = 0 → 循环结束，返回 left = 1

left 指向了 1，这正是 2 应该插入的位置。这并非巧合——left 始终指向第一个不小于 target 的元素的位置，也就是插入位置。

再试一个边界：target = 0。

  1. left = 0, right = 3, mid = 1, nums[1] = 3 > 0 → right = 0
  2. left = 0, right = 0, mid = 0, nums[0] = 1 > 0 → right = -1
  3. left = 0, right = -1 → 循环结束，返回 left = 0

正确，0 应该插在开头。

再来：target = 7。

  1. left = 0, right = 3, mid = 1, nums[1] = 3 < 7 → left = 2
  2. left = 2, right = 3, mid = 2, nums[2] = 5 < 7 → left = 3
  3. left = 3, right = 3, mid = 3, nums[3] = 6 < 7 → left = 4
  4. left = 4, right = 3 → 循环结束，返回 left = 4

正确，7 应该插在末尾。

所以结论是：二分查找结束后的 left 就是我们要的答案。

循环条件：while (left <= right) 还是 while (left < right)？

我们用的是 left <= right，对应闭区间 [left, right]。当 left == right 时，区间里还有一个元素，需要继续判断。如果写成 <，就会漏掉这个元素，导致需要在循环外额外处理。

用 <= 是最直接、最不容易出错的写法。

复杂度分析

版本 时间复杂度 空间复杂度 说明
暴力扫描 $O(n)$ $O(1)$ 一次遍历，最坏情况扫描整个数组

延伸思考

这道题让我们学到了什么？

排序数组 + 查找问题 ≈ 二分查找。 这是一个条件反射式的联想。只要题目给出”排序”或”有序”这个条件，同时又涉及查找，二分查找就应该是第一个跳进脑海的方案。

此外，这道题也帮我们理清了二分查找的循环不变量的重要性。选择什么样的区间定义（闭区间、左闭右开……），边界怎么移动，返回值是什么——这些都取决于我们在一开始定下的循环不变量。一旦定好，全程保持一致，代码就不会出错。

下次遇到类似问题优先观察什么？

  1. 数组是否有序？ 有序就优先考虑二分查找或其变体。
  2. 是否要求 O(log n)？ 看到对数级别的复杂度要求，基本就是在提示二分查找或分治。
  3. 如果找不到，要返回什么？ 像本题这样要求返回插入位置的，往往是二分查找结束时 left 指针的位置。

掌握了二分查找的思路，下次遇到”在排序数组中查找第一个大于等于 target 的元素”这类问题（比如 LeetCode 34、278），我们也能够举一反三，快速写出正确的代码。
