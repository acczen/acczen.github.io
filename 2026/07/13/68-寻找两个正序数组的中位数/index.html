寻找两个正序数组的中位数

寻找两个正序数组的中位数

核心思路

首先，最简单的方法是什么？

题目给了我们两个已经排好序的数组，让我们找出它们合并后的中位数。

最直接的办法——把两个数组合并成一个有序数组，然后直接取中间位置的值：

这种方法的时间复杂度是 O(m+n)，空间复杂度也是 O(m+n)。虽然能通过，但题目明确要求时间复杂度为 O(log(m+n))——看到 log，我们自然联想到二分查找。

不合并数组，能直接找到中位数吗？

让我们思考一下：中位数的本质是什么？

中位数将一个有序序列分成左右两个等长的部分，且左半部分的所有元素都不大于右半部分的所有元素。

也就是说，我们要在合并后的序列中找到一个分割点，使得左右两侧元素个数相等（或左侧多一个）。

既然两个数组都是有序的，这个分割点必然同时落在两个数组上。换句话说，我们需要在 nums1 中选一个分割位置 i，在 nums2 中选一个分割位置 j，使得：

  1. 左侧元素总数 = 右侧元素总数（或左侧多一个）
  2. 左侧的最大值 <= 右侧的最小值

这样一来，中位数就由分割点两侧的元素决定。

怎么找到这个分割点？

假设 nums1 的长度为 m，nums2 的长度为 n。合并后的序列总长度为 m+n。

如果我们把合并后序列的左侧定义为包含前 (m+n+1)/2 个元素（向下取整），那么左侧的元素个数就是固定的。

设 nums1 左侧取了 i 个元素，则 nums2 左侧必须取 j = (m+n+1)/2 - i 个元素，这样两侧总数才匹配。

现在问题转化为：找到一个 i（0 <= i <= m），使得：

  • nums1[i-1] <= nums2[j] —— nums1 左侧的最大值不超过 nums2 右侧的最小值
  • nums2[j-1] <= nums1[i] —— nums2 左侧的最大值不超过 nums1 右侧的最小值

如果这两个条件都满足，我们就找到了正确的分割！

为什么用二分？

i 的取值范围是 [0, m]，我们可以暴力尝试每一个 i，但这样是 O(m)。不过仔细观察会发现一个重要性质：

如果 nums1[i-1] > nums2[j]，说明 nums1 左侧取多了，i 应该减小（让 nums1 少贡献一些元素，nums2 多贡献一些）。

如果 nums2[j-1] > nums1[i]，说明 nums1 左侧取少了，i 应该增大。

这就构成了二分的条件！我们可以在 nums1 上对 i 进行二分搜索。

还有一个关键优化：我们总是在较短的数组上进行二分。因为二分的时间复杂度是 O(log m)，我们希望 m 尽可能小，所以取 nums1 为较短的数组。这样整体复杂度就是 O(log(min(m, n)))。

边界情况如何处理？

二分过程中，i 可能为 0（nums1 左侧没有元素）或 m（nums1 右侧没有元素），j 也可能为 0 或 n。我们需要用特殊值来处理：

  • 当 i = 0 时，nums1[i-1] 不存在，视为 -∞
  • 当 i = m 时，nums1[i] 不存在，视为 +∞
  • 当 j = 0 时，nums2[j-1] 不存在，视为 -∞
  • 当 j = n 时，nums2[j] 不存在，视为 +∞

这样比较条件仍然可以统一处理。

找到分割后如何计算中位数？

  • 如果 (m+n) 是奇数，中位数就是左侧的最大值：max(nums1LeftMax, nums2LeftMax)
  • 如果 (m+n) 是偶数，中位数是左侧最大值和右侧最小值的平均：(max(leftMax) + min(rightMin)) / 2

延伸思考

这道题让我们学到了什么？

理解中位数的定义比套模板更重要。

这道题最妙的地方在于：我们不是直接”找”中位数，而是通过中位数的定义——“将序列分为等长的左右两部分，且左 <= 右”——反过来构造出了分割方案。

很多二分问题都是类似的：不是在一维数组上直接搜索目标值，而是搜索一个满足条件的位置。条件判断函数决定了二分的走向。

“在短数组上二分”是一个重要技巧。

当问题涉及两个数组且需要 O(log) 时间复杂度时，通常是对较短的数组进行二分，这样可以保证 log 的参数尽量小。这是一种”降维”思维：把二维的搜索问题转化为一维的二分搜索。

下次遇到类似问题优先观察什么？

  1. 看到 O(log) 的时间复杂度要求，立刻想到二分。
  2. 两个有序数组 + 中位数 + O(log)，大概率是分割 + 二分。
  3. 先问自己：能不能直接对短数组进行二分？ 如果能找到单调性条件（比如某个值偏大需要减小分割点），二分就可行。
  4. 边界处理是这类题目的难点——提前想好 i=0, i=m, j=0, j=n 四种情况，用 ±∞ 统一处理，往往比写 if-else 分支更清晰。

这道题是经典的”hard 题 soft 解”——核心思想并不复杂，难的是把边界情况考虑周全。理解了分割 + 二分的思路，以后遇到类似问题（比如在两个有序数组中找第 k 小的数），就能举一反三了。
