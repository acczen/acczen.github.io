LeetCode 560. subarray-sum-equals-k

给你一个整数数组 nums 和一个整数 k，请你统计并返回该数组中和为 k 的连续子数组的个数。

提示：

  • $1 \le \text{nums.length} \le 2 \times 10^4$
  • $-1000 \le \text{nums}[i] \le 1000$
  • $-10^7 \le k \le 10^7$

核心思路

拿到这道题，我们最容易想到的是什么？

没错——枚举所有子数组，把每个子数组的和算出来，看看有几个等于 $k$。双重循环就能搞定：

这个解法的时间复杂度是 $O(n^2)$。在 $n$ 高达 $2 \times 10^4$ 的情况下，$O(n^2)$ 大约是 $4 \times 10^8$ 次操作，一定会超时。

瓶颈在哪里？ 对于每个起点 $i$，我们都要重新扫描一遍后面的元素来累加求和。这些累加过程中有大量的重复计算。

那有没有办法”记住”之前算过的信息，避免重复累加？

想象一下：如果我们知道从数组开头到某个位置 $i$ 的累积和（记作 $\text{preSum}[i]$），那么任意子数组 $[l, r]$ 的和就可以表示为：

$$\text{sum}(l, r) = \text{preSum}[r] - \text{preSum}[l - 1]$$

也就是说，子数组求和问题可以转化为两个前缀和之差。

现在问题变成了：我们要找有多少对 $(l, r)$ 满足 $\text{preSum}[r] - \text{preSum}[l-1] = k$。

移项一下：$\text{preSum}[l-1] = \text{preSum}[r] - k$。

这意味着什么？当我们遍历到位置 $r$ 时，前缀和是 $\text{preSum}[r]$。我们只需要知道在位置 $r$ 之前，有多少个前缀和等于 $\text{preSum}[r] - k$。

这就像「两数之和」那道题——只不过我们把”两数之和等于 target”变成了”两前缀和之差等于 k”。而「两数之和」我们用哈希表在 $O(1)$ 时间内就能查询某个值出现的次数。

到这里，思路就清晰了：一边遍历一边用哈希表记录每个前缀和出现的次数，对于当前位置，查一下之前有多少个前缀和等于 $\text{currentPreSum} - k$，累加到答案中。

解法：前缀和 + 哈希表

我们遍历数组，维护一个变量 preSum 表示从数组开头到当前位置的累积和。同时用一个哈希表 map 记录每个前缀和值出现的次数。

具体步骤：

  1. 初始化 map，放入 {0: 1}——表示前缀和为 0 出现了 1 次。这为什么重要？考虑子数组从索引 0 开始的情况：如果 preSum == k，说明从开头到当前位置的整段子数组和为 $k$，此时 $preSum - k = 0$，map 中必须有一个 0 才能统计到这种情况。
  2. 遍历数组，更新 preSum += nums[i]。
  3. 检查 map 中是否存在 preSum - k，如果存在，把对应的出现次数加到答案里。
  4. 把当前的 preSum 记录到 map 中（次数 +1）。
  5. 遍历结束后，答案即为所求。

为什么用哈希表而不是其他结构？我们需要快速查询某个值出现的次数，哈希表提供 $O(1)$ 的查找和更新，完美匹配这个需求。前缀和的值可能为负且范围很大（$-2 \times 10^9$ 到 $2 \times 10^9$），用数组存不下，所以哈希表是自然之选。

与暴力解法的对比：

方案 时间复杂度 空间复杂度 核心思想
暴力枚举 $O(n^2)$ $O(1)$ 枚举所有子数组并求和
前缀和 + 哈希表 $O(n)$ $O(n)$ 将子数组和转为前缀和之差，用哈希表 $O(1)$ 查询

我们用 $O(n)$ 的额外空间换来了从 $O(n^2)$ 到 $O(n)$ 的时间优化，在 $n = 20000$ 时，这是从”跑不完”到”毫秒级”的质变。

复杂度分析

维度 分析
时间复杂度 $O(n)$ —— 只需遍历数组一次，每次哈希表操作均为 $O(1)$
空间复杂度 $O(n)$ —— 最坏情况下每个位置的前缀和都不同，哈希表存储 $n$ 个键值对

延伸思考

学到了什么？

这道题的核心技巧是将子数组求和转化为前缀和之差，再借助哈希表把「两数之差」问题降为 $O(n)$。这是一类非常经典的模板——凡是涉及”连续子数组满足某种和条件”的题，前缀和 + 哈希表往往是第一个值得尝试的方向。

下次遇到类似问题，你会优先观察什么？

优先观察能不能把「子数组的和 / 异或 / 乘积」转化为「两个前缀值的关系式」，然后问自己：这个关系式能不能用一个哈希表在遍历过程中 $O(1)$ 查询？如果可以，$O(n)$ 解法就呼之欲出了。
