环形链表

环形链表

核心思路

第一反应：怎么判断一个链表有没有环？

先看问题本身——给定一个链表，判断它是否有环。

直观地理解，”环”意味着链表中的某个节点的 next 指针指回了之前已经访问过的节点，导致遍历永远不会结束。

那么最朴素的想法是什么？如果我们遍历链表，每到一个节点就看看之前是不是来过——这不就是”标记法”吗？

暴力思路：用哈希表记录访问过的节点

我们遍历链表，每遇到一个节点，就在哈希表中查一下它是否出现过：

  • 如果出现过，说明有环，返回 true；
  • 如果没有出现过，就把它加入哈希表，继续往下走；
  • 如果走到了 null，说明链表无环，返回 false。

这个思路很自然，也很容易实现。时间复杂度 O(n)，空间复杂度 O(n)。

但问题来了——能否不用额外的空间，在 O(1) 空间内解决？

能不能不用额外空间？

哈希表需要 O(n) 的空间。如果链表很长（比如几百万个节点），这个空间开销就不小了。

那我们怎么在不记录”历史”的情况下判断有没有环呢？

不妨想一下环的特性——在环上绕圈，意味着同样的节点会被反复经过。

有没有什么办法，让两个指针一快一慢地在链表上跑，通过它们是否相遇来判断有没有环？

这正是经典的 Floyd 判圈算法，也叫”快慢指针”或”龟兔赛跑”算法。

优化：快慢指针

思路是这样的：

  • 让慢指针（slow）每次走一步，快指针（fast）每次走两步。
  • 如果链表没有环，快指针会率先走到 null，此时返回 false。
  • 如果链表有环，快指针会在环里一圈一圈地追慢指针——因为快指针比慢指针每次多走一步，所以从相对运动的角度看，快指针相对于慢指针以”每步一个节点”的速度靠近它，最终一定会相遇。

为什么一定会相遇？假设慢指针刚进入环的入口时，快指针已经在环内的某个位置。设环的长度为 L，快指针相对于慢指针的速度差是 1 步/次，那么最坏情况下，快指针需要追赶的距离不超过 L，所以最多 L 步后两者必然相遇。

这个算法的时间复杂度仍然是 O(n)，但空间复杂度降到了 O(1)。

一个小细节：为什么快指针每次走两步，不是三步或更多？

不妨想一下——如果快指针每次走三步，相对速度是 2 步/次，追赶速度更快，看似更早相遇。但有一个隐患：快指针可能会”跳过”慢指针。在环的长度和起始位置的一些组合下，快指针可能永远在慢指针前面一个节点、两个节点……但就是不相遇吗？

实际上，只要速度和步长是整数且两者步长差为大于 0 的整数，就一定会在有限步内相遇——详细证明需要用到模运算。但”每次走两步”是最简单、最直观、不会引起额外疑虑的选择，所以实践中统一用两步。

边界情况

  • 空链表（head == null）：直接返回 false。
  • 只有一个节点且无环：fast 在第一步就走到 null，返回 false。
  • 只有一个节点且指向自身（环）：slow 和 fast 都指向这个节点，初始时 fast.next == head，进入循环后第一次移动就相遇了，返回 true。

复杂度分析表

版本 时间复杂度 空间复杂度 说明
哈希表 O(n) O(n) 遍历一次链表，哈希表最多存储 n 个节点
快慢指针 O(n) O(1) 无环时 fast 遍历到末尾；有环时 fast 和 slow 在环内相遇，总步数不超过 O(n)

两种方法的时间复杂度都是 O(n)。在有环的情况下，快慢指针的循环次数不会超过链表总长度与环的长度之和，仍为线性级别。但快慢指针的空间优势是巨大的——只用了两个指针变量，没有额外数据结构。

延伸思考

这道题让我们学到了什么？

  1. 双指针的妙用——一快一慢两个指针在同一数据结构上以不同速度移动，可以解决很多与”环”、”中点”、”倒数第 k 个”相关的问题。快慢指针不只适用于链表，也适用于数组（比如判断重复数）等场景。

  2. 空间换时间和时间换空间的权衡——哈希表思路直接、易于理解，但需要 O(n) 空间。快慢指针需要一点思维上的跳跃，但节省了空间。在面试中，先给出哈希表解法，再优化到快慢指针，展示了从”可工作”到”更优”的思考过程。

  3. 对链表结构的理解——链表不像数组可以通过下标随机访问，但通过指针操作可以做出很灵巧的算法。理解链表的引用传递特性，是解决这类问题的基础。

下次遇到类似问题优先观察什么？

  • 问题是否涉及”循环””重复””相遇”这些概念——这往往是快慢指针的信号。
  • 空间限制是否严格——如果要求 O(1) 空间，大概率是要用双指针或原地操作。
  • 链表长度是否未知——如果不知道长度，快慢指针比哈希表更稳健，因为不需要预分配空间。
  • 是否需要定位环的入口——如果进一步要求返回环的起始节点，快慢指针扩展一下（找到相遇点后，让一个指针从头开始，另一个从相遇点继续，每次都走一步，再次相遇处即为环入口）就可以解决，这也是一种常见的 follow-up 问题。

从环形链表想到的……

环形链表的检测是很多链表进阶问题的基础。比如：

  • 环形链表 II（142）：找到环的入口节点。
  • 快乐数（202）：虽然是数字问题，但检测循环的本质和环形链表完全一样——可以用快慢指针来判断是否会无限循环。
  • 寻找重复数（287）：给定一个数组，里面的数字在 1 到 n 之间，找到那个重复的数。把数组下标和值看作链表节点的 next 指针，问题就转化成了找环的入口。

这些题目表面不同，但核心都是”在一个会循环的结构中检测/定位循环”，而快慢指针就是解开这类问题的钥匙。

让我们记住这种感觉——面对一个”无限走下去”的问题时，不妨想一想：有没有两个不同速度的指针，能帮我们判断这条路上有没有绕不出去的圈子？
