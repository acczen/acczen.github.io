LeetCode 11. 盛最多水的容器（container-with-most-water）

题目

LeetCode 11. container-with-most-water

给定一个长度为 $n$ 的整数数组 height。有 $n$ 条垂直线，第 $i$ 条线的两个端点是 $(i, 0)$ 和 $(i, height[i])$。

找出其中的两条线，使得它们与 $x$ 轴共同构成的容器可以容纳最多的水。返回容器可以储存的最大水量。

注意：你不能倾斜容器。

提示：

  • $n = height.length$
  • $2 \le n \le 10^5$
  • $0 \le height[i] \le 10^4$

核心思路

拿到这道题，我们不妨先问问自己：如果没有任何优化技巧，最朴素的做法是什么？

很自然地，我们会想到枚举所有可能的左右边界组合 $(i, j)$，计算它们围成的容器面积 $\min(height[i], height[j]) \times (j - i)$，然后取最大值。

这个思路对不对？对的。但问题出在哪？

如果 $n = 10^5$，枚举所有组合需要多少次计算？

组合数大约是 $\frac{n(n-1)}{2} \approx 5 \times 10^9$ 次，显然会超时。所以，暴力解法在思路上是正确的，但在数据规模面前不够用。

那么关键问题是：我们真的需要检查每一对组合吗？能不能跳过一些明显不可能成为答案的组合？

我们来观察一下容器的面积公式：

$$
\text{Area} = \min(height[left], height[right]) \times (right - left)
$$

这里有三个变量：左边界高度、右边界高度、以及宽度。直觉上，宽度越宽越有利，高度越高越有利。但两者是矛盾的——我们无法同时最大化宽度和高度。

再追问一步：如果我们从两端（宽度最大）开始，然后逐步向中间移动，会发生什么？

当我们把左右指针放在数组两端时，宽度是最大的。接下来如果要移动指针，宽度一定会变小。那么，要想让面积变大，唯一的希望就是高度变得更大。

现在问自己一个关键问题：

假设 $height[left] < height[right]$，我们应该移动哪个指针？

容器的实际高度由较矮的那根决定。如果我们移动较高的指针（right），新容器的高度不可能超过 $height[left]$（因为短板还是 left 或者更矮），而宽度却减小了——面积只可能变小或不变。

反之，如果我们移动较矮的指针（left），虽然宽度也减小了，但新高度有可能超过原来的 $height[left]$，从而有机会得到更大的面积。

这就是贪心的直觉：每次移动较矮的那一侧，赌一个更高的”墙”来弥补宽度的损失。

这个思考链条下来，我们从”枚举所有”自然过渡到了”只检查 $O(n)$ 对候选”——这就是双指针解法。

解法：双指针（对撞指针）

算法步骤

  1. 初始化左右指针 $left = 0$，$right = n - 1$。
  2. 计算当前容器面积 $\min(height[left], height[right]) \times (right - left)$，更新最大值。
  3. 比较 $height[left]$ 和 $height[right]$：
    • 如果 $height[left] < height[right]$，则 $left$ 右移一步；
    • 否则 $right$ 左移一步。
  4. 重复步骤 2-3，直到 $left \ge right$。
  5. 返回记录的最大值。

为什么选择双指针？

与其他备选方案的对比：

方案 时间复杂度 空间复杂度 可行性
暴力枚举 $O(n^2)$ $O(1)$ $n=10^5$ 时超时
单调栈 / / 本题不适用——面积由两根柱子共同决定，不是单侧极值问题
排序 + 二分 $O(n\log n)$ $O(n)$ 排序会丢失下标信息，宽度无法计算
双指针 $O(n)$ $O(1)$ 最优

双指针的正确性基于一个非常漂亮的不变量：被淘汰的那些组合，不可能成为最优解。当我们移动较矮的指针时，我们实际上是在说”以当前较矮柱子为边界的、更窄的那些容器，都不需要看了”——因为宽度更小、高度又不会超过这根矮柱子，面积必然更小。这个”剪枝”动作扔掉的是严格劣解，所以不会错过最优解。

实现注意事项

  • 指针移动条件：height[left] < height[right] 时移动左指针，否则移动右指针。注意这里用了 < 而非 <=，两者在正确性上等价，但 < 让相等时移动右指针——其实移动哪边都可以，因为两根一样高时，无论移动哪边，新容器的高度都不会超过当前的高度。
  • 面积计算：小心不要写成 height[left] * height[right]——这是矩形面积，不是容器面积。正确的公式是 $\min(height[left], height[right]) \times (right - left)$。
  • 数据范围：$height[i]$ 最大 $10^4$，$n$ 最大 $10^5$，理论上最大面积约为 $10^4 \times 10^5 = 10^9$，在 Java int 范围内（约 $2.1 \times 10^9$），直接用 int 即可。但如果未来数据加强，记得考虑 long。

复杂度分析

维度 分析
时间复杂度 $O(n)$ — 左右指针各自从两端向中间移动，每个元素最多被访问一次，共 $n$ 步
空间复杂度 $O(1)$ — 只用了几个变量记录指针位置和最大值，不随输入规模增长

延伸思考

学到了什么？

这道题的核心思维模式是利用贪心策略，通过排除严格劣解来缩小搜索空间。很多看似需要 $O(n^2)$ 枚举的问题，只要我们能证明”某些候选解一定不如另一些候选解”，就可以用双指针或滑动窗口将复杂度降到 $O(n)$。关键不在于记住”移动矮的那一根”这个结论，而在于理解为什么被跳过的那些组合不会影响最终答案。

下次遇到类似问题，你会优先观察什么？

先问自己：问题的最优解是否具有”单调舍弃”的性质？ 换句话说，如果我按照某种顺序遍历候选解，能否在每一步证明”不看某些候选也不会错过最优解”？如果能找到这样的不变量，双指针、滑动窗口或者贪心大概率就是突破口。这类问题的典型信号包括：两数之和类问题中的有序性、接雨水、三数之和等。
