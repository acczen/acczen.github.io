二叉树的最近公共祖先

二叉树的最近公共祖先

核心思路

问题理解

题目给了我们一棵二叉树和两个节点 p、q，要求找到它们最近的（即深度最大的）公共祖先。

所谓”祖先”，是指一个节点在树中能够向上追溯到根的所有节点——包括它自己。”公共祖先”就是既包含 p 又包含 q 的节点。”最近”意味着深度最大，也就是离这两个节点最近的公共祖先。

不妨先想一个最直接的思路。

暴力解法：记录祖先路径

假设我们能看到每个节点到根节点的路径，那问题就变得非常简单了：

  1. 从根出发，找到 p 的路径：比如 [3, 5, 6, 2]
  2. 从根出发，找到 q 的路径：比如 [3, 5, 7]
  3. 从后往前（或从根往叶子）找到两条路径的最后一个公共节点：5 就是 LCA

这个思路很符合直觉，但需要两次遍历来收集路径，还要额外 O(N) 空间来存路径。能否做得更优雅？

自问自答：能不能一次性搞定？

问题： 如果我们只遍历一次树，有什么办法能在遍历过程中直接判断出 LCA？

想想看，当我们站在某个节点 node 上时，我们真正关心的是什么？

关键观察： 对于节点 node，如果 p 和 q 分别出现在它的左子树和右子树中，那么 node 就是 LCA。如果 p 和 q 都在左子树，那么 LCA 在左子树里；都在右子树，LCA 在右子树里。

这暗示我们：需要先知道左右子树的情况，才能判断当前节点是不是 LCA——这正是后序遍历（左右根）的特征。

递归的直觉

让我们站在一个普通节点 node 的视角，用递归思维来思考：

问自己： 我作为当前节点，怎么判断子树里有没有 p 或 q？

答案很简单：如果当前节点本身就是 p 或 q，那我的子树里肯定包含目标。如果左子树说”我找到了”，右子树也说”我找到了”，那当前节点就是 LCA。

  • 从根 3 开始：左子树找 5 返回节点 5，右子树找 1 返回节点 1
  • 左右都不为空 → 返回 3
  • 所以 LCA = 3

找 p = 7、q = 4 的 LCA：

  • 从根 3 开始递归到 5，再到 2
  • 节点 2：左子树 7 返回 7，右子树 4 返回 4
  • 左右都不为空 → 返回 2
  • 所以 LCA = 2

延伸思考

这道题让我们学到了什么？

1. 后序遍历的威力

这道题最核心的洞察是：要判断当前节点是不是 LCA，必须先知道左右子树的情况——这就是后序遍历的天然应用场景。很多二叉树问题（比如求树的直径、最大路径和）都遵循这个模式：先处理子树，再用子树信息决定当前节点。

2. 递归返回值的多重含义

我们的递归函数返回值其实承载了两种不同的语义：它要么返回一个找到的目标节点（p 或 q），要么返回找到的 LCA。这种”一值多用”的技巧在树递归中非常常见。

3. 自底向上的思维

遇到树的问题，不妨问自己：如果站在叶子节点，我能知道什么？如果站在某个中间节点，我需要从子树获取什么信息才能做出判断？这种自底向上的思维方式往往能帮我们找到递归解。

下次遇到类似问题优先观察什么？

  • 问题是否涉及”祖先”或”公共祖先”？考虑递归+后序遍历
  • 是否可以自底向上传递信息？考虑每个节点返回什么值给父节点
  • 什么时候能确定答案？当左右子树各返回一个非空结果时
  • 如果树是二叉搜索树（BST）呢？可以利用大小关系进一步优化，但思路本质相同
  • 如果允许存储父指针呢？可以用哈希表记录每个节点的父节点，然后从 p 向上标记路径，再从 q 向上查找第一个被标记的节点——但这是另一种 trade-off

进一步挑战

如果题目改成”任意多节点的最近公共祖先”，上面的递归解法只需稍作改动即可适用——把 p 和 q 的判等改为在一个集合中查找即可。这说明递归后序遍历的思路具有很强的泛化能力。
