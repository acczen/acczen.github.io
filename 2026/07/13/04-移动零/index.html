LeetCode 283. 移动零（move-zeroes）

题目

LeetCode 283. move-zeroes

给定一个数组 nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。

请注意：必须在不复制数组的情况下原地对数组进行操作。

提示：

  • $1 \le \text{nums.length} \le 10^4$
  • $-2^{31} \le \text{nums[i]} \le 2^{31} - 1$

进阶：你能尽量减少完成的操作次数吗？

核心思路

拿到这道题，我们不妨先问自己一个问题：如果不考虑”原地操作”这条约束，最简单的做法是什么？

是不是很自然——创建一个新数组，先遍历一遍原数组，把所有非零元素依次塞进去，然后再把剩余的位置全部填上 0。这样当然是对的，时间复杂度 $O(n)$，只是空间复杂度也是 $O(n)$，因为我们”偷懒”开了一个新家。

现在，题目偏偏说”必须原地操作“。这就像一个整理房间的场景：你不能把东西全搬出去重新摆，只能在现有的房间里挪来挪去。

这就引出一个关键问题：在只有一个房间的情况下，我们怎么知道每个东西该放到哪？

想一想，我们在乎的东西是非零元素，而零是我们要”赶到末尾”的对象。如果用一个指针 j 始终标记”下一个非零元素应该放的位置”，然后我们用另一个指针 i 从头到尾扫描——每遇到一个非零元素，就把它和 j 位置交换，同时 j 向后挪一步。扫描结束时，j 之后的所有位置都正好是我们想要的零。

这个思路就叫双指针（更准确地说是快慢指针或读写指针）。它之所以有效，是因为 j 始终指向数组中”已被整理好的区域”的末尾后一个位置——这恰好也是数组中第一个零的位置。

思考链条回顾：暴力开新数组 $O(n)/O(n)$ → 瓶颈在于额外空间 → 原地操作意味着我们需要”就地标记” → 引入一个写指针跟踪非零元素的落点 → 双指针一趟遍历 $O(n)/O(1)$。

解法：双指针（一次遍历交换）

为什么是双指针？

面对”原地、保持顺序、分离两类元素”的题目，双指针几乎是条件反射式的解法。这道题与「移除元素」（LeetCode 27）、「删除有序数组中的重复项」（LeetCode 26）共享同一套思维框架——它们都在做同一件事：维护一个”有效区间”，把符合条件的元素往前提，不符合的往后推。

算法步骤

我们定义两个指针：

  • 快指针 i：负责扫描整个数组，寻找非零元素。
  • 慢指针 j：始终指向”下一个非零元素应该被放置的位置”，也是当前第一个零的位置。

具体过程：

  1. 初始化 $j = 0$，表示第一个非零元素应该放在索引 0。
  2. 用 i 从 0 扫描到数组末尾：
    • 如果 $nums[i] \neq 0$，说明找到了一个需要前移的元素。
    • 交换 $nums[i]$ 和 $nums[j]$——这个操作把非零元素放到 j 位置，同时原来的元素（要么是 0，要么是 $i = j$ 时的自己）被换到后面。
    • $j$ 后移一位，继续指向下一个空位（第一个 0）。
  3. 扫描结束，所有非零元素被挪到前面，零自然地留在后面。

你可以这样想象：j 就像一条”已整理区域”的分界线，j 左边全是非零元素且保持原序，j 及其右侧是待处理区域。每次 i 发现非零元素，就把它”塞进”已整理区域的末尾，分界线右移。

为什么交换而不是覆盖？

你可能见过另一种写法：先覆盖再补零（两遍遍历）。交换法更好，因为它一趟遍历完成所有操作，而且代码更简洁。当 i == j 时交换实际上是和自己交换，虽然不影响正确性，但可以加一个 if (i != j) 跳过——这在面试中提一句就是加分项。

复杂度分析

维度 分析
时间复杂度 $O(n)$ — 每个元素被 i 访问一次，交换操作是 $O(1)$，没有嵌套循环
空间复杂度 $O(1)$ — 只使用了两个指针变量，没有额外数组或递归栈

这是本题的渐近最优解——要移动所有元素，必须至少遍历一遍数组，$O(n)$ 是时间下界。

延伸思考

学到了什么？

双指针的核心不在于”两个指针”本身，而在于用一个指针维护”已处理区间”的边界。这个思维模式是数组类题目的高频考点，一旦内化，面对「移除元素」「删除重复项」「颜色分类」这些题就会觉得似曾相识。

下次遇到类似问题，你会优先观察什么？

当题目同时出现**”原地操作”、“保持相对顺序”、“分离两类元素”**这三条中的任意两条时，第一时间问自己：能不能用一个慢指针维护有效区边界，让快指针去扫描？ 这几乎就是双指针模板的信号灯。
