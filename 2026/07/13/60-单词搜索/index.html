单词搜索

单词搜索

核心思路

先从一个朴素的想法开始

题目给了我们一个 m x n 的二维网格和一个单词，让我们判断单词是否存在于网格中。所谓”存在”，是指单词的每个字母在网格中按顺序相邻（上下左右）出现，且同一个格子不能重复使用。

让我们先想一想最直观的做法是什么。

我们可以枚举所有可能的路径吗？

网格中的每个格子都可以作为起点。从某一个起点出发，我们需要沿着四个方向依次匹配单词的每个字母。这其实就是在一张图上做”行走”，每一步走到相邻的格子，并且不能重复走同一个格子。

那这本质上是什么问题呢？图的遍历。具体来说，是一个带约束的深度优先搜索——约束是：已走过的格子不能再走，且路径上的字母必须按顺序匹配目标单词。

暴力搜索会面临什么？

不妨想一下：对于一个 m x n 的网格，假如我们从 (0,0) 出发，每次有最多 4 个方向可以选择（边界处更少），单词长度为 L，那么最坏情况下，搜索空间有多大？

每一层最多分支 4 个方向，一共需要深入 L 层，所以搜索空间是 O(m * n * 4^L)。这里 m * n 是起点的数量。

这个复杂度听起来挺大，但实际中因为字母匹配条件的约束和边界的限制，大多数分支在很浅的深度就会被剪掉。所以暴力 DFS 在实践中往往是可行的。

如何用代码实现 DFS 回溯？

DFS 沿着一条路径走到黑，如果发现走不通，就退回到上一个状态尝试另一个方向——这个”退回”的过程就是回溯的核心思想。

具体来说，我们用函数 dfs(board, word, i, j, k) 表示：当前在格子 (i,j)，正准备匹配单词的第 k 个字符。函数的返回值是布尔值，表示从当前状态能否找到剩余的单词。

那怎样标记格子已访问过呢？这里有一个常用的技巧：直接在原数组上标记。把当前格子字符暂存起来，替换成一个特殊字符（比如 # 或 \0），DFS 返回后再恢复原值。这样就避免了额外的 visited 数组，空间更省，代码也更简洁。

让我们走一遍流程：

  1. 遍历网格的每个格子作为起点。
  2. 从该格子调用 DFS，尝试匹配整个单词。
  3. 在 DFS 中：
    • 如果 k == word.length()，说明所有字符都已匹配成功，返回 true。
    • 如果当前格子越界、字符不匹配，或格子已经被访问过（值为 #），返回 false。
    • 标记当前格子为已访问（设为 #）。
    • 向四个方向递归搜索。
    • 回溯：恢复当前格子的原始字符。
    • 如果任一方向返回 true，则当前 DFS 返回 true。

还有什么可以优化的吗？

一个很实用的剪枝技巧：在开始 DFS 之前，先做一个”入场检查”。

技巧一：长度检查。 如果单词长度大于网格中字母的总数，直接返回 false。

技巧二：频率检查。 统计网格中每个字符出现的频率，与单词中每个字符的频率进行比较。如果单词中出现了网格中没有的字符，或者某个字符的需求量大于网格的供应量，直接返回 false。

技巧三：反转单词。 这是一个不那么直观但很有效的优化。如果单词第一个字符的出现频率比最后一个字符的出现频率更高，我们把单词反转过来搜索。为什么呢？因为起点越多，DFS 分支越多。如果让更稀有的字符作为起点，就能在早期大幅减少分支数量。

这三个优化虽然不改变最坏情况下的复杂度，但在实际测试中能显著提速，尤其是遇到大网格和长单词时。

复杂度分析

版本 时间复杂度 空间复杂度 说明
基础回溯 $O(m \cdot n \cdot 4^L)$ $O(L)$ $m$ 和 $n$ 是网格尺寸，$L$ 是单词长度。空间来自递归栈深度。
带剪枝回溯 最坏同上，实际大幅降低 $O(L)$ 频率检查和反转单词不影响渐进复杂度，但能大幅削减实际搜索空间。

关于时间复杂度的进一步说明：

$4^L$ 这一项让人有些紧张，但实际上由于字母匹配的限制，有效的分支远少于 4 个。比如当单词是 "abcd"，而网格中大部分区域都是字母 a 时，第一层分支会很多，但第二层开始绝大多数分支就会被字母不匹配剪掉。

如果把搜索过程理解为一棵树的遍历，那么这棵树的有效分支因子往往远小于 4，接近 1 甚至更低。

关于空间复杂度：

我们直接在原数组上标记访问状态，没有使用额外的 visited 数组，所以空间开销仅有递归调用栈的深度 $O(L)$。Java 中将 String word 转为 char[] 也是一个小优化，避免每次调用 charAt()。

延伸思考

这道题让我们学到了什么？

回溯法是在探索”所有可能”与”及时止损”之间寻找平衡。

当我们面对一个”是否存在一条路径”的问题时，第一反应就应该是 DFS + 回溯。这是最直接、最自然的思维模型。

更重要的是，一个小小的剪枝能带来巨大的实际收益。反转单词这个技巧尤其有趣——它告诉我们，在搜索问题中，优先处理”选择更少”的分支，是通用的优化原则。这和我们后面会遇到的各种搜索问题（比如数独、N 皇后）中的”最少候选优先”策略是一脉相承的。

另外，在原数组上直接标记代替 visited 数组是一个值得记住的小技巧。它不光省空间，代码也更紧凑。缺点是会修改输入数据——如果题目不允许修改原数组，那就需要另辟蹊径了。

下次遇到类似问题优先观察什么？

下一次在二维网格上看到”是否存在一条路径”类的题目时，不妨按这个顺序思考：

  1. 能否抽象为图的遍历？ 网格天然就是一个图，四个方向就是四条边。
  2. DFS 还是 BFS？ 需要走完一条完整路径时，DFS 更自然（递归的回溯机制天然适配路径探索）。BFS 更适合最短路径。
  3. 如何标记已访问？ 优先考虑原地标记，同时注意题目是否允许修改。
  4. 能提前剪枝吗？ 长度、频率、反转——这三板斧在任何搜索问题里都可以先试一遍。
  5. 路径的唯一性约束——“同一个格子不能重复使用”这个约束保证了搜索不会陷入无限循环，但也意味着我们需要维护一个”记忆”机制。

这道题是回溯法的经典入门题，它不涉及复杂的优化技巧，但完整地展示了回溯法的骨架：选择 → 递归 → 撤销选择。把这个骨架牢记在心，后面遇到更复杂的搜索问题就容易多了。
