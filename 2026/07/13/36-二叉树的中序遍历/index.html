二叉树的中序遍历

二叉树的中序遍历

核心思路

第一反应：递归，有什么好想的？

给定一棵二叉树，输出它的中序遍历序列。中序遍历的定义每个学二叉树的人都背过：左子树 -> 根节点 -> 右子树。

但且慢——如果面试官说”不许用递归”呢？

递归虽然简洁，但它的本质是系统帮我们维护了一个隐式的函数调用栈。当树的高度很大时（比如一棵链状树有 10000 个节点），递归会导致栈溢出。这是我们在生产代码中必须考虑的问题。

那么，不用递归，我们如何实现中序遍历？

不用递归，那就手动模拟栈

递归的好处是系统自动帮我们管理了”回溯”的时机——当我们遍历完左子树后，系统栈自动把当前节点弹出来，让我们访问它，然后继续遍历右子树。

如果我们自己用显式的栈来模拟这个过程，是不是就能绕开递归的限制？

不妨想一下这个过程：

  1. 从根节点出发，一路向左走，把沿途经过的节点都压入栈中。
  2. 当走到 null 时，从栈中弹出一个节点——这个节点就是当前最左的节点，访问它。
  3. 然后转向该节点的右子树，重复步骤 1。
  4. 直到栈为空且当前节点为 null，遍历结束。

这其实就是迭代版的中序遍历，用我们自己的栈代替了系统栈。

有的朋友可能会问：为什么前序遍历的迭代写法比中序遍历简单？ 因为前序遍历访问根节点的时机恰好是我们第一次见到它的时候，而中序遍历需要先憋着，等左子树全部遍历完再访问根——这个”等一等”的动作，就需要靠栈来记忆了。

还能再优化吗？Morris 遍历

迭代版用栈把空间复杂度降到了 O(n)（最坏情况下栈里存了整棵树），但有没有办法做到 O(1) 的空间？

你可能觉得：遍历树总要记录路径吧，怎么可能 O(1)？

这里有一个非常巧妙的想法——利用叶子节点的空指针来记录回溯路径。这就是 Morris 遍历的核心思想。

树中有大量的 null 指针（左右孩子为空的指针域），Morris 巧妙地利用了这些空闲指针来建立一种”线索”，让我们在遍历完左子树后能自动回到根节点，而无需额外用栈存储。

具体来说，Morris 中序遍历的思路是：

  1. 从根节点开始，记当前节点为 cur。
  2. 如果 cur.left == null，说明左子树为空，直接访问 cur，然后令 cur = cur.right。
  3. 如果 cur.left != null，我们需要找到 cur 在中序遍历中的前驱节点，也就是左子树中最右边的节点（记为 pre）。
    • 如果 pre.right == null，说明我们还没处理过这棵左子树，就把 pre.right 指向 cur（建立线索），然后 cur = cur.left 继续向左。
    • 如果 pre.right == cur，说明左子树已经遍历完了，我们通过线索回到了 cur，此时应该恢复 pre.right = null（断开线索，还原树的原貌），访问 cur，然后 cur = cur.right。

这听起来有点绕，但核心就一句话：把左子树最右侧节点的空右指针，临时指向根节点，这样遍历完左子树后就能自然地回到根节点。

Morris 遍历的厉害之处在于：

  • 空间复杂度 O(1)，只用了几个指针变量
  • 时间复杂度仍然是 O(n)，每个节点最多被访问两次（一次建立线索，一次断开线索）

当然它也有代价——代码的理解难度和编写复杂度比前两种方案高不少，而且在实际工程中，大多数场景递归或栈已经够用了。Morris 更适合那些对内存极度敏感的场景（比如嵌入式环境），或者用来在面试中展示你对树的深刻理解。

延伸思考

这道题让我们学到了什么？

遍历的本质是”顺序”问题。 前序、中序、后序只是访问根节点的时机不同——前序是”第一次见到就访问”，中序是”从左子树回来后才访问”，后序是”从右子树回来后才访问”。理解了这一点，三种遍历就都能用同一套递归或迭代模板搞定。

递归的简洁背后隐藏了系统栈。 递归之所以简单，是因为语言 runtime 替我们管理了调用栈。当我们想摆脱递归时，本质上就是用我们自己的数据结构（显式栈）来接管这份工作。

空间是可以”借”来的。 Morris 遍历的巧妙之处在于利用了已有的空指针来传递信息——这是一种”借空间”的思想。在算法设计中，当我们觉得空间不够用时，不妨想一想：现有的数据中有没有闲置的存储位置可以利用？

下次遇到类似问题优先观察什么？

  • 优先判断能否递归。 如果题目没有限制空间或明确要求非递归，递归往往是最快最稳的写法。不要为了炫技而选择复杂的方案。
  • 如果限制非递归，显式栈法是普适方案。 二叉树的前序、中序、后序遍历都可以用栈模拟——只是入栈和访问的顺序不同而已。中序遍历是”先憋着，左子树走完再访问”。
  • 如果限制 O(1) 额外空间，想想 Morris。 Morris 遍历的三个步骤可以背诵，但更重要的是理解”线索”的思想——用空闲指针建立临时路径，用完即还原。这个思路不仅适用于中序遍历，也适用于前序和 Morris 变体。

最后留一个问题给我们自己思考：Morris 后序遍历怎么做？ 它比中序更复杂一些，但思路是一脉相承的——感兴趣的朋友可以动手画一画树，看看后序的 Morris 版本和线索的建立时机有什么不同。
